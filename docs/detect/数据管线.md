# 检测数据管线全流程（获取 → 组装 → 训练消费）

> ADR-0006 重写后的**当前**检测数据管线。回答「一张图/一批 raw 怎么变成 NanoDet 能吃的训练集，
> 且每张图可追溯、每步可复现」。数据集**设计**（5 类体系、开源盘点、负样本策略）看
> [01-数据集.md](01-数据集.md)；训练/评估口径看 [02-训练与评估.md](02-训练与评估.md)；踩坑实录看
> [03-实操日志.md](03-实操日志.md)。设计决策见 [[ADR-0003]]（adapter seam）/[[ADR-0004]]（5 类）/
> [[ADR-0006]]（获取标准化 + 溯源透明 + JSONL）。

## 0. 一句话

**两阶段：`acquire`（获取，可联网/上 box）→ `build`（组装，离线纯净）。** 每个数据源 = 一个自包含
可插拔 `DatasetAdapter`，声明「从哪来、怎么映射、许可如何」；加新源 = 写一个 adapter + 注册，
组装/训练 caller 零改（[[ADR-0003]]）。

## 1. 全流程图

## 2. 声明层：DatasetAdapter + DatasetSpec

每个源在 `src/edge_cam/data/adapters/detect/<name>.py` 里声明一个 `DatasetSpec`（代码即单一事实源）：

| 字段 | 作用 |
|---|---|
| `label_map` | 源类目名 → 5 类名（未列入 = 丢弃该类目） |
| `license` / `commercial_safe` / `role` | 许可(SPDX) · 是否可商用 · train / eval_only |
| `exhaustive` | 穷尽标注(缺类当负样本)/非穷尽(未标区域 ignore，防漏标污染) |
| `split_unit` | image / location / sequence（相机陷阱按组防泄漏） |
| `negative_quota` / `max_per_class` | 负样本限额 / 每类含图上限（治长尾） |
| **`acquire: AcquireSpec`** | **method + urls + version + archive_sha256** —— 数据从哪来（[[ADR-0006]] D1） |

`FEEDER5_CATEGORIES`（bird/squirrel/cat/person/other_animal）是 **contracts 层规范**，打标契约与
adapter 共用（单一来源，防漂移）。**加新源 = 写 adapter（声明 spec + 实现 `load_raw` + 可选 `_fetch`）
+ `register_adapter`**，caller 零改。

## 3. ① 获取阶段（`acquire`）

两阶段的头。`build` 假设 raw 已就位、保持离线纯净；获取从「手动 + 散脚本 + prose」收敛为**一处标准命令**。

```bash
# 全源来源清单（不下载）：license / role / method / version / URL 一处看全
python -m edge_cam.data.adapters.detect.acquire --list

# 按 build config 里选中的源逐源获取（--source 只跑一个；--dry-run 只算不下）
python -m edge_cam.data.adapters.detect.acquire --config configs/data/detect_feeder5.yaml
```

- `adapter.acquire(raw_root, now)` 按 `AcquireSpec.method` 分派；`now` 由调用方传入（时间戳纯函数化）。
- **method = manual**（LILA/COCO 暂无自动下载器）：校验 raw 就位 + `archive_sha256`，缺失**抛可执行错误
  （错误信息直接给下载 URL）**，不静默放行。
- **method = s3_direct / roboflow / inat_open_data**：子类覆写 `_fetch()` 实下载；基类统一校验 + 收据；
  **幂等**（已校验通过则跳下载）。
- 每次落 **`_acquire.json` 收据**（source/method/urls/version/archive_sha256/downloaded_at/counts）→
  「这批 raw 怎么来的」可复现、可审计。
- raw 落盘布局：`raw_root/<commercial|feasibility>/<name>/`（layer 进路径 → 筛商用 = glob commercial/*）。

### 3b. MegaDetector 伪标注（iNat 等无框源专用，独立 GPU 阶段）

iNat 是分类图**无 bbox** → 走独立阶段（**不在** `acquire()` 内，避免基类拖 GPU/pytorch-wildlife）：

```
iNat 图 → MegaDetector(MDV6-mit/apa，隔离 env) → COCO(框) → (可选)Label Studio 人审
```

- 框记 `label_provenance`：`md_pseudo`（MD 自动）→ 人审通过升 `md_human_verified`（**信任分层**，透明可审）。
- **先量 MD 可信度**：在有框源（ENA24/CCT）上跑 `eval/megadetector.py` 得 MD 的 AP/bird 召回 → 定信任阈值。
- **分层 QA（不全人审，保规模）**：MD 高置信自动收 / 中置信 Label Studio 人审 / 低置信丢。
- MD 权重不进产物、不发行、只出框坐标（非版权物，不传染）；伪标注是把 MD 知识迁进小 student 的**蒸馏**载体。

## 4. ② 组装阶段（`build`）

```bash
python -m edge_cam.data.adapters.detect.build --config configs/data/detect_feeder5.yaml
```

1. **`load_raw()`**（源特定解析）→ `RawSample`（源标签未映射 + 逐样本 author/url/media_id/sha256 +
   框级 label_provenance）。COCO json / 合并多 split（Roboflow、CCT）/ 相机陷阱按 location 取分组键，
   各源自理。
2. **基类 `build_records()`**（族无关公共逻辑）：
   - 映射源标签 → 5 类，丢未映射；
   - 负样本：0 框图仅**穷尽源**或**显式 `is_negative`**才保留，受 `negative_quota` 限额；
   - `max_per_class` 每类含图限额（int 全类同上限 / dict 只压指定类，不误伤欠采样类）；
   - **确定性 split**：`_split_of(group_key, seed)` 按内容 key 哈希 → 同 group（location/sequence）必同
     split，防泄漏；
   - **attribution 流**：`RawSample` 的逐样本溯源写入 `DetImageRecord` + 框 `label_provenance`。
3. **`assemble()`** 按 role/commercial 路由：
   - `train` = 可商用 & role=train 的 train+val 切；`test` = 同批 test 切；
   - `eval_feasibility` = role=eval_only（COCO，仅评估，绝不进训练）；
   - **§4 硬闸**：role=train 但 `commercial_safe=False` → 直接 `raise`（不可商用数据不得进训练）。

## 5. 产物（`data/processed/detect/`）

| 产物 | 内容 |
|---|---|
| `manifest_{train,test,eval_feasibility}.jsonl` + `.meta.json` | **JSONL 唯一格式**（逐行一 DetImageRecord + meta sidecar：name/version/categories/root）—— 可流式、可 diff、规模友好（[[ADR-0006]] D5） |
| `labels/{key}_{split}.json` | NanoDet 吃的 COCO labels（`to_coco` 派生视图；provenance/split 仍由 manifest 统一承载不丢） |
| `license_manifest.csv` | **7 列逐图署名**：path/source/license/author/original_url/source_media_id/asset_sha256 → 真兑现 CC-BY（§4，此前只 source+license 不足） |
| `summary.json` | 各 split 计数 |

## 6. ③ 训练消费

NanoDet（`NanodetBackend`，box）吃 `labels/*.json`（COCO）；DetectionManifest 的 provenance/split 一路
承载不丢，最终经 `provenance_from_manifest` 流进 ModelCard → registry（许可红线全链路可追溯）。训练口径
见 [02-训练与评估.md](02-训练与评估.md)。

## 7. 横切关注点

- **5 类体系**（[[ADR-0004]]）：bird 单一类（细分交分类器），长尾哺乳动物并 other_animal（治稀有类饿死）。
- **溯源透明链**（§4）：每图 → author/url/license/source/asset_sha256 → `license_manifest.csv` +
  `_acquire.json` 收据 → ModelCard → registry。**全链路可追溯**。
- **防泄漏 split**：相机陷阱按 location/sequence 整组进同一 split；网图按 image。
- **负样本**：检测器天生前景/背景判别 → 空/落叶/非类 = 不出框 = 不触发；Caltech empty 帧当主力负样本。
  详见 [01-数据集.md](01-数据集.md) §7。
- **可插拔契约**：7 源全走同一 `acquire --list`；加源不改 caller。
- **信任分层**：框 `label_provenance ∈ {gt, md_pseudo, md_human_verified}` → 训练可按信任加权/分阶段。

## 10. 加一个新源（可插拔契约）

1. 写 `data/adapters/detect/<name>.py`：声明 `DatasetSpec`（含 `label_map` + `acquire`）+ 实现
   `load_raw()`（多为 `CocoJsonAdapter` 子类）+ 需自动下载则覆写 `_fetch()`；`register_adapter`。
2. `data/adapters/detect/__init__.py` import 该模块（注册副作用）。
3. `configs/data/detect_feeder5.yaml` 的 `datasets:` 加一行（+ 组装期 overrides）。
4. 上 box `acquire`（拿 raw + 校验）→ `audit_unmapped()` 核 label_map → `build`。**组装/训练 caller 零改。**

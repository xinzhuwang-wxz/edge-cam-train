# 02 · 粗检测训练与评估

> 接 [01-数据集.md](01-数据集.md)。本文定:模型变体、预训练微调、数据增强、训练安排、评估口径、
> MegaDetector 角色、设备指标。原则:**bird 召回为先、框完整(给分类器好 crop)、上板算子安全**。

## 1. 模型变体（都 NanoDet-Plus-m，ShuffleNetV2，NPU 安全）

| 变体 | 参数 | FLOPs | COCO mAP | 角色 |
|---|---|---|---|---|
| **nanodet-plus-m 320 1.0x** | 1.17M | 0.9G | 27.0 | **部署默认**(最轻上板) |
| nanodet-plus-m 416 1.0x | 1.17M | 1.52G | 30.4 | 分辨率臂(测高分辨率换小鸟召回，实验1 用过) |
| nanodet-plus-m-1.5x 320 | 2.44M | 1.75G | 29.9 | 容量臂(测更大模型增益) |

控制变量消融：分辨率(320 vs 416)× 容量(1.0x vs 1.5x)。INT8 ~980KB / FP16 1.8MB。
**不选**：YOLOX-Nano(Focus/slice，§4 NPU 回退软算子)、PicoDet(ESNet 的 SE/h-swish，实验1 已证量化不友好)。详见 [README](README.md)。

## 2. 预训练与微调

- **起点 = COCO 预训练**(NanoDet-Plus-m 官方 ckpt)。**全量微调**：加载 backbone+FPN(neck)，**头因 80→5 类重置**，所有层一起训(实验1 同法)。
- **单阶段**(本轮)：只用 commercial-clean 公开数据微调。两阶段(公开适配 → 自有领域微调)**待自有数据**再做(避免灾难性遗忘时再混采公开数据)。
- 优化：AdamW + cosine，参考实验1(50–100 ep / batch 视显存)。

## 3. 数据增强（固定机位场景，慎用强几何）

| 适合 | 慎用(喂食器机位固定，别造不存在的场景) |
|---|---|
| 亮度/对比度/饱和度、轻微模糊、JPEG 压缩、随机遮挡、雨雪/水滴模拟、灰度/夜视模拟、轻微缩放平移、水平翻转 | 强 Mosaic、大角度旋转、严重透视、把动物缩到不现实尺寸 |

## 3b. 类不均衡处理（5 类长尾，框架已支持）

数据天然不均（实测 other_animal 远多于 bird/squirrel）。三层杠杆，**优先数据侧**（可复现、跨框架）：

1. **数据装配侧（主力，DatasetAdapter）**——组装时下采样过多的类：
   - `max_per_class`：`int`（所有类同一上限）或 `dict{类名: 上限}`（**只压指定类、不动欠采样类**）。
     例：`max_per_class: {other_animal: 12000}` 只压 other_animal，bird 全留。
   - `negative_quota`：控负样本数量。两者都按 path 哈希确定性抽样。
2. **分类训练侧**（细分类器，`configs/train/classify/config.yaml`）——二选一：
   - `optim.class_weighted: true` → 反频类权重喂 `CrossEntropyLoss(weight=)`。
   - `data.balanced_sampler: true` → `WeightedRandomSampler` 反频过采样。
3. **检测训练侧（NanoDet）**：NanoDet-Plus 的 **GFL/Quality Focal Loss + 动态 assigner** 天生对
   前景/背景不均衡鲁棒；5 类间不均衡主要靠①数据侧 `max_per_class`，必要时再调它 config 的 loss 权重
   （我们不 fork，属进阶）。检测**不用**图级过采样（一图多类，语义不清）。

> 落地顺序：先全源 build 看真实分布 → 调 `max_per_class` 压 other_animal 到 ~bird 量级 → 训练。

## 4. 评估口径（按优先级，承实验1 教训:框质量 >> mAP）

1. **bird 召回(第一优先)**:Recall@conf={0.10,0.20,0.30}。目标:白天 ≥95%、困难集 ≥85%。宁可少量误报，别漏珍贵鸟访。
2. **bird 误判(混淆矩阵)**:squirrel→bird、cat→bird、leaf/insect→bird、empty→bird，单独输出。
3. **定位质量(给分类器好 crop)**:AP50、Bird AP50、Bird Recall@IoU50、**裁剪完整率**(框别切头/翅/尾)。此场景 AP50 比 AP50:95 更贴业务；裁剪时框外扩 10–20%。
4. **场景分组**:白天/夜视IR、大/小目标、完整/局部身体、单/多鸟、清晰/运动模糊、北美/欧洲、公开/(未来)真机。
5. **额外评估集**:除 test，还在 **feasibility 集(COCO/NABirds)** 上评(泛化 + bird 召回信号)。
6. **设备指标(待板子，V85x/V851S 真机)**:NPU 延迟、CPU 后处理延迟、端到端延迟、峰值内存、模型大小、功耗、INT8 掉点、每小时误触发。

**模型选择评分(建议)**:40% bird 召回 / 20% squirrel→bird 误判 / 15% 困难召回 / 10% ROI 完整率 / 10% 真机延迟 / 5% 内存+大小。**不因快 3ms 接受明显更高漏鸟率。**

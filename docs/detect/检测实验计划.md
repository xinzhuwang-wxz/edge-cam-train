# 检测实验计划 · Round 1（零样本基线 + 2 路微调 bake-off）

> 经 grill 定稿（对话 2026-07）。主线 = 可行性优先（[[ADR-0001]]）。本轮**在带框数据上**回答两个
> **硬决策** + 一条基线，为 round 2（微调 + 补 feeder/iNat）铺路。数据管线见
> [数据管线.md](数据管线.md)；模型选型背景见 [02-训练与评估.md](02-训练与评估.md) + 开发笔记 §7.1。

## 1. 目标与产出（这轮"决定"什么）

**两个硬决策 + 基线**：

| 产出 | 怎么得到 | 判据 |
|---|---|---|
| **① MD 能不能当教师**（firm） | 在带框源上评 MegaDetector 的框准 | **bird 召回@IoU0.5 ≥ 0.90 且 bird 框 AP50 ≥ 0.60** → 才敢用它给 round 2 的 iNat 伪标注 |
| **② round 2 用哪个骨干**（firm） | **2 路 mini bake-off**：RTMDet-tiny vs NanoDet-1.5x 在 ENA24+CCT 短训 → held-out 上比 | 微调后 **bird 召回 + mAP** 谁高谁上（RTMDet 明显更强则换、打平则留 NanoDet 省事） |
| **基线锚点** | 零样本全模型 + feeder 客串 | 量"开箱起点"与"微调头顶空间" |

**不是部署 go/no-go**——本轮不部署任何东西；量化这轮**完全不做**（性能优先，`int8` 留后）。

## 2. 范围（这轮做 / 不做）

- **做**：零样本推理（COCO 预训练 + MD + feeder 客串）+ 2 路微调 bake-off；全 **FP32**；全在**带框**数据上。
- **不做**：新源（Roboflow feeder / iNat）→ round 2；量化/INT8；全 5 路 bake-off（只 2 路定向）；上板。

## 3. 模型与权重

| 配置 | params | 权重 | 框架 | 本轮角色 |
|---|---|---|---|---|
| NanoDet-Plus-m 320 | 1.2M | COCO 预训练 | NanoDet | 零样本 |
| NanoDet-Plus-m 416 | 1.2M | COCO 预训练 | NanoDet | 零样本 |
| NanoDet-Plus-m 1.5x-320 | 2.4M | COCO 预训练 | NanoDet | 零样本 + **bake-off** |
| RTMDet-tiny | 4.8M | COCO 预训练 | MMDetection | 零样本 + **bake-off** |
| RTMDet-s | 8.9M | COCO 预训练 | MMDetection | 零样本 |
| feeder_320 / 416 | 1.2M | **已微调 5 类**（旧管线） | NanoDet(ONNX) | **客串锚点** |
| MegaDetector | — | 域预训练（300 万相机陷阱图） | pytorch-wildlife | **教师候选** |

- **权重现实**：NanoDet/RTMDet 官方**只有 COCO**（RTMDet 的 Objects365 只是骨干预训练，输出头仍 COCO-80）→ 零样本能检 bird/cat/person，**认不得 squirrel**。真正"贴域"的预训练检测器就是 **MegaDetector**（已在实验内）。
- **客串泄漏坑**：feeder 当年训在 ENA24+CCT → 它在这两源的数是"**见过的**"（偏乐观），**只有 COCO val 是它的公平泛化数** → 客串按源读、ENA24/CCT 标注乐观。

## 4. 数据（全带框；见 [数据管线.md](数据管线.md)）

| 源 | 域 | 本轮角色 | 许可 |
|---|---|---|---|
| **ENA24** | 相机陷阱(东北美) | bake-off 训练 + eval | CDLA 可商用 |
| **Caltech CT**(ECCV18 sm) | 相机陷阱(美西南) + empty | bake-off 训练 + eval + **负样本** | CDLA 可商用 |
| **COCO 2017 val** | 网图 | 零样本 eval + feeder 客串"干净"评测 | eval_only（不进训练，§4） |

- **不进**：OIV7（带框可商用但下载重；bake-off 2 路排名用不着，默认不加；要 bird 更丰富可选加）· Roboflow/iNat（round 2）。
- **split**：ENA24+CCT 切 train/val/test，**test 比例调大到 ~30%**（本轮不训最终模型，held-out 越大指标越稳）→ **零样本模型、bake-off 模型、feeder 客串都在同一 `test` + COCO 上评**，数字才可比。JSONL 记录 durable，round 2 拓展后重切零成本（`_split_of` 确定性）。
- **产物**：`manifest_{train,val,test}.jsonl` + `manifest_eval_feasibility.jsonl`(COCO) + `labels/*.json` + `license_manifest.csv` + `summary.json`。

## 5. Harness（一切 → FP32 ONNX → onnxruntime → 统一评测）

- **前向统一**：每模型 → FP32 ONNX，`onnxruntime` 跑前向（免装各家推理 env）。
- **解码每族不同**（ONNX 是裸 backbone+head，§4）：NanoDet 解码**已有**（级联 `OnnxDetector` numpy）；**新写 RTMDet numpy 解码**；MD 走其 ONNX/pytorch-wildlife。
- **统一评测**：所有预测 → COCO 格式 → **泛化版 `eval/megadetector.py`**（把"GT 折叠 + COCOeval + bird 召回"从 MD 专用泛化成任意检测器）→ 一张 `detect_ablation` 表。
- **类映射**（COCO-80 → 我们的对比口径）：
  - bird→bird、cat→cat、person→person、dog→other_animal；COCO **无 squirrel**（零样本 squirrel≈0）。
  - MD：animal→{bird/squirrel/cat/other_animal 折叠}、person→person。
- **bake-off 训练**：`RtmdetBackend`（MMDet subprocess 包一层，对等 `NanodetBackend`，[[ADR-0003]]）+ 装 MMDet env。

## 6. 指标与判据（bird 为先）

1. **bird 为先（产品 #1）**：bird 召回@conf{0.10,0.20,0.30}、bird AP50。
   - COCO 检测器：GT-bird 被预测的 **COCO-bird** 命中；MD：GT-bird 被**任意 animal 框**命中。
2. **any-animal（跨模型公平）**：全部输出 + GT 折叠 animal/person → "找到 critter"召回 + AP50。
3. **盲区显式列**：COCO 检测器的 squirrel / other_animal 召回（预期≈0）→ 量化"不微调的代价"。
4. IoU 阈固定 **0.5**；按**源分组**报（域不同，别混）。
5. **MD 教师阈值**：见 §1 表①。

## 7. 两阶段流程

**Phase 1 · 零样本（便宜，先跑）**
1. 全模型 ONNX + MD + feeder 客串 在 `test`+COCO 上推理 → 统一评测 → **谱表**。
2. 出 **MD 裁决**（阈值①）+ **去风险**（每个 arch 的 ONNX/解码是否 work、能否找鸟）——可提前否决 bake-off 候选。

**Phase 2 · 2 路 bake-off（要 GPU）**
3. RTMDet-tiny vs NanoDet-1.5x 在 ENA24+CCT `train`+`val` 短训（~30-50 ep）。
4. 在**同一 `test`+COCO** 上评微调后 bird 召回/mAP → **骨干拍板②**。

## 8. 算力与租卡前后分工（AutoDL 无卡模式）

| 阶段 | 态 | 做什么 | 吃 GPU |
|---|---|---|---|
| 租卡前（本地/仓库） | — | 备 harness 代码（RTMDet 解码 / 泛化评测 / RtmdetBackend / ONNX runner / build 实验 config）+ 本计划。**全可本地单测** | 否 |
| ① 下载+build | **无卡模式**(¥0.1/hr) | `acquire` 下三源（学术加速）→ `build` 出 JSONL + split | 否 |
| ② 导 ONNX | 无卡模式 | NanoDet×3 / RTMDet-tiny·s → FP32 ONNX；MD 取 ONNX；feeder 已有 | 否 |
| ③ Phase 1 零样本 | 无卡/小 GPU | 推理 + 统一评测 + MD 裁决 | 小（ONNX 可 CPU；MD 宜 GPU） |
| ④ Phase 2 bake-off | **切 GPU**(3080 档) | 2 路短训 + 评 | **要** |

**租卡前"数据准备好"的准确含义**：代码+config 本地就位；**raw 图与 JSONL 在无卡模式产**（下载必须在 box）。所以严格"纯本地准备好 JSONL"做不到——**无卡模式下载+build 就是那一步**，几乎不花钱。

## 9. 消融表 schema（`detect_ablation`）

行 = **模型 × 源**（ENA24 / CCT / COCO 分组）；列：

`model · source · bird_recall@0.1 · @0.2 · @0.3 · bird_AP50 · anyanimal_recall@0.5 · anyanimal_AP50 · squirrel_recall · other_recall · note(泄漏/域标注)`

## 10. 报告形态

一张**谱表**（零样本-COCO < MD < 微调，见谱）+ **MD 教师裁决**（过/不过阈值①）+ **骨干拍板②**（RTMDet vs NanoDet 微调对比）+ **round 2 建议**（补 feeder/iNat + MD 伪标注 + 选定骨干微调 5 类）。落 `results/detect/round1_zeroshot/`。

## 11. Round 2 展望（本轮铺路）

选定骨干 + MD（若过阈）当教师 → 补 Roboflow feeder(域) + iNat(覆盖，MD 伪标注 + Label Studio 人审)
→ 微调 5 类 → 真正的可行性包络 + gate + 发布链（数据 split 那时重切）。

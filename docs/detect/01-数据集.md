# 01 · 粗检测数据集方案

> 目标:用开源 + (未来)自建数据,训出 5 类喂鸟器粗检测器。本文定:类别体系、数据集盘点、
> license 分层、DatasetAdapter 抽象、存放结构、组装与清洗。决策见 [[ADR-0004]]（5 类）、[[ADR-0003]]（adapter）。

## 1. 类别体系（5 类，[[ADR-0004]]）

| idx | 类 | 含义 / 产品动作 |
|---|---|---|
| 0 | `bird` | 目标；框 → crop → 细分类器出种 |
| 1 | `squirrel` | 头号害兽/混淆源；单列 |
| 2 | `cat` | 捕食者 |
| 3 | `person` | 补粮的人；检到→抑制告警 |
| 4 | `other_animal` | 长尾哺乳动物合并(raccoon/rabbit/fox/dog/deer/opossum/rodent…)，治样本饿死 |

- bird 在检测层是**单一类**（细分交分类器）。`empty`/无框图 = **负样本**（不是类别）。
- 实验1 用 11 类，skunk AP 1.3、raccoon 42（稀有类饿死）→ 本方案合并长尾。

## 2. 开源数据集盘点

| 数据集 | 内容 | raw 格式 | 怎么用 | 许可（§4） | → 5 类 |
|---|---|---|---|---|---|
| **Open Images V7** | 网图，Bird/Squirrel/Cat/Person/Raccoon/Rabbit/Fox/Deer… 带框 | OIV7（FiftyOne 按类拉）→COCO | FiftyOne 按 class 下载子集 | 逐图 **CC-BY**（署名清册）→**可商用** | 全 5 类主力 |
| **ENA24-detection**(LILA) | 东北美 ~10k 图**全带框**，23 类 | COCO JSON | 直接用，物种→5类 | **CDLA-Permissive→可商用** | squirrel/other/bird，域好 |
| **Caltech Camera Traps**(LILA) | 美西南 244k 图 / ~66k 框，22 类，夜视IR/遮挡/真实 | COCO JSON + 图在 GCS/S3 | 筛 bbox 子集 + empty 负样本 | **CDLA-Permissive→可商用** | other_animal 长尾 + bird + 负样本 |
| **Roboflow 喂食器/松鼠** | **喂食器域最对口**(bird-feeder、squirrel garden) | COCO 导出 | API/导出，**逐个核 license** | 多 CC-BY/Public（逐个核）→可商用 | bird/squirrel 域最真 |
| NACTI / Missouri CT(LILA) | 相机陷阱带框子集 | COCO JSON | 取 bbox 子集 | CDLA→可商用 | bird/other 补充 |
| **COCO 2017** | 网图 bird/cat/dog/person 框 | COCO JSON | 抽含目标子集 + 负样本 | 图杂→**仅可行性** | （仅评估）bird/cat/person |
| NABirds / CUB-200 | 鸟细类带框(48k/11k) | 自有→COCO | bird 框增强 | **研究用→仅可行性** | （仅评估）bird 增强 |

来源:[LILA bbox 清单](https://lila.science/megadetector-results-for-camera-trap-datasets/) · [ENA24](https://lila.science/datasets/ena24detection) · [Caltech CT](https://lila.science/datasets/caltech-camera-traps) · [Roboflow bird-feeder](https://universe.roboflow.com/bird-recognition/bird-feeder) · [NABirds/CUB](https://www.vision.caltech.edu/datasets/cub_200_2011/)

## 3. license 分层（决定进训练还是仅评估）

§4 海外发行从严 + 传染规则（含 NC/研究数据 → 权重不可商用）：

| 层 | 数据集 | 用途 |
|---|---|---|
| **commercial**（可商用） | OIV7 / ENA24 / Caltech-CT / Roboflow(permissive) / NACTI / Missouri | **进训练** → 权重天生可商用 |
| **feasibility**（仅可行性） | COCO / NABirds / CUB | **仅评估**（泛化/bird 召回信号），**绝不进训练** |
| **self**（自建） | 自有喂食器（未来） | 进训练（MegaDetector 伪标注后） |

- 训练集 = commercial(+self) 的并集 → 无传染、可发行。
- 可行性数据照常准备(走同一 adapter 管线)，标 `commercial_safe=False`、`role=eval_only`。
- 逐图 `attribution`(作者/URL/license) → `license_manifest.csv` 随产物披露(OIV7/Roboflow 商用要求)。

## 4. DatasetAdapter 抽象（[[ADR-0003]] 落地，每数据集自包含）

每个数据集 = 一个**自注册 adapter**，声明：
```
name / raw_format(coco_json|fiftyone_oiv7|roboflow) / raw_root
label_map      {源标签 → 5 类}（None = 丢弃该标签）
license / commercial_safe / role(train | eval_only)
exhaustive     非穷尽(OIV7)→ ignore-policy；穷尽(ENA24)→ 缺类当负样本
split_unit     image | location | sequence（相机陷阱按地点/序列防泄漏）
max_per_class  每类抽样上限（控不平衡，bird 不淹没 other_animal）
negative_quota 拉多少无框负样本（empty/no-target）；Caltech 高、OIV7 中、ENA24≈0（全标注）
attribution    逐图 作者/URL/license（商用署名清册）
clean          dedup(sha256 + pHash) + min_box_size + 连续帧去重
→ 产出 DetImageRecord[]（带 provenance + role）
```
注册表(`adapters/detect/base.py` 的 `register_adapter`)汇总所有 adapter → 组装。**加新源 = 写一个
adapter 注册，组装/训练/评估 caller 不改。** 代码位置：`src/edge_cam/data/adapters/detect/<dataset>.py`。

### 已落地（[[ADR-0003]] 深基类 + 薄 adapter）

| 模块 | 角色 | 关键事实（经联网核验真实标注 json） |
|---|---|---|
| `base.py` | 抽象 + 组装 | `DatasetSpec`/`RawSample`/`DetectionDatasetAdapter`；`build_records`（映射→5类、丢未映射、负样本、`negative_quota`/`max_per_class` 限额、确定性 split、provenance）；`assemble`（按 role/commercial 路由 train/test/eval_feasibility，role=train&不可商用直接 raise）；`build_adapter(name,raw_root,**ov)` |
| `coco_json.py` | 共用解析 | COCO instances JSON 通用（ENA24/CCT/COCO/NACTI/Roboflow 导出）；`audit_unmapped()` 上线前复核映射 |
| `ena24.py` | commercial | 23 类全带框 `exhaustive=True`；**无 location/seq → 按 image 划分**；CDLA-Permissive |
| `caltech_ct.py` | commercial | bbox 子集 + 图级 json 合并：join `location`/`seq_id`（**按 location 划分**防泄漏）、补 `empty` 空帧当负样本；CDLA-Permissive |
| `fiftyone_oiv7.py` | commercial | FiftyOne 按类拉；`exhaustive=False`；`Mouse`(动物)≠`Computer mouse`；CC-BY + 逐图署名 |
| `coco2017.py` | feasibility | `role=eval_only`/`commercial_safe=False`，仅评估不进训练 |
| `build.py` | 入口 | config → 各 adapter → assemble → `manifest_*.json` + NanoDet labels + `license_manifest.csv` |

组装：`python -m edge_cam.data.adapters.detect.build --config configs/data/detect_feeder5.yaml`。
> 各数据集类目字符串以**实际 json 为准**：上 AutoDL 拿到 raw 后先跑 `adapter.audit_unmapped()` 复核 label_map。

## 6. 组装与清洗

- **组装**：训练集 = `commercial_safe & role=train` 的 DetImageRecord 并集；评估集 = 各 adapter 的 eval split + 全部 feasibility adapter。
- **清洗**(每 adapter 自带策略，merge_map 提供算子)：
  - 去重：SHA-256 完全去重 + pHash/dHash 近似去重 + 连续视频帧去重。
  - 非穷尽源(OIV7)：未标 target 类区域 → ignore（不当负样本，§5.1 防漏标污染）。
  - 最小框门控：太小/几像素 → ignore（不当明确负/正）。
- **数据量目标**(第一轮 30k–50k 图，类框均衡)：bird 20–30k 框 / squirrel 8–15k / cat·person 各 3–5k / other_animal 5–10k / 负样本 10–20k 图。`max_per_class` 控 bird 不淹没长尾。

## 7. 负样本与困难负样本（树叶/石头/昆虫怎么处理 → 不需要"是不是动物"前置）

**检测器天生做前景/背景判别**：只在有把握有目标时出框，树叶/石头/空 = 背景 = 不出框。
所以**不需要**第 6 个"背景"类或前置"动物分类器"（冗余 + 变两阶段检测，边缘不划算）。
树叶/石头/昆虫靠**喂负样本**解决。

> ⚠️ **现状（实测）：实验1 检测数据 0 负样本**（train 15000 / val 3335 **全部有框**）——
> 因旧 ingest 用 `load_zoo_dataset(classes=…)` 只拉含目标的图。检测器从没见过"啥都没有"的图 →
> 易对树叶/空喂食器误报。**本轮必须显式补负样本**（见下"来源"+ adapter 的 `negative_quota`）。

**负样本从哪来（明确来源 + 配额）：**

| 类型 | 来源 | 许可 | 量 |
|---|---|---|---|
| 相机陷阱 empty（主力，域真实） | **Caltech CT 的 `empty` 帧**（含植被/光照/夜视，最像真实背景） | CDLA→可商用 | 8k–15k |
| 无目标网图（补充） | OIV7/COCO 中**不含 5 类**的图 | OIV7 可商用 / COCO 仅可行性 | 3k–5k |
| **困难负样本（价值最高）** | 树叶/树枝晃动、昆虫贴镜头、水滴/蛛网、强逆光过曝、运动模糊 —— **最易误报成 bird** | 自有(未来)为主；现可挖 Caltech 含植被/反光的 empty | 越多越好 |
| 隐式负样本 | 正样本图框外的背景区域（每张含鸟图也在教"框外不是鸟"） | 随正样本 | 免费 |

- **机制落地**：每个 adapter 声明 `negative_quota`（Caltech 高、OIV7 中、ENA24≈0 因全标注）→ 组装时按配额拉 empty/no-target 图（无框）。目标负样本 **10k–20k 图（占 20–40%）**。
- **双保险**：推理 **置信阈值** + 级联 **置信门控**（[02](02-训练与评估.md) / CascadePipeline）：低置信框滤掉/回退，不把树叶 crop 喂种分类器。
- 旁证：MegaDetector 即"animal/person/vehicle vs blank"检测器，靠海量 empty 学滤背景，我们同构。

> 评估侧对应：混淆矩阵单列 `leaf/insect→bird`、`empty→bird`（见 [02](02-训练与评估.md)）。

## 8. split（防泄漏，关键）

**不能随机按图划分**。相机陷阱同一地点/同一事件序列的连帧高度相关 → 必须**整组进同一 split**。
- split_unit：相机陷阱按 `location`/`sequence`；网图(OIV7/COCO)按 `image`。
- 比例 train 70 / val 15 / test 15。
- 额外测试集：**Public Test**(公开数据未见地点，泛化)、(未来)**Feeder Test**(自有未见设备，量产决策主集)。

## 9. 数据获取与存储

**各数据集大小（筛选后实际要下的）：**

| 数据集 | 全量 | 筛选后 |
|---|---|---|
| COCO 2017 | ~19GB | bird/cat/person 子集 ~2–5GB |
| Open Images V7 | 500GB+ | 按类 + max_samples ~10–20GB |
| ENA24 | ~8GB | 全要 ~8GB |
| Caltech CT | ~105GB | bbox 子集 + empty ~30–40GB |
| NACTI | ~1.4TB | bbox 子集 ~3GB |
| Roboflow / MegaDetector 权重 | 小 | ~1–2GB / ~150MB |
| **合计** | | **~60–90GB** |

→ 超笔记本舒适区，**下到 AutoDL 整理**（大盘）。两个坑：

1. **国外源限速**：COCO/OIV7/LILA 均国外，AutoDL 拉国外慢(~1MB/s)。对策：① 后台 `screen` 挂下；
   ② 用对工具(LILA 在 AWS/GCP/Azure → gsutil/aws-cli/azcopy 优于 wget；OIV7 用 FiftyOne；COCO 找国内镜像)；
   ③ **annotation-first**：先下标注 JSON(小) → 筛 image-id → 只下需要的图(省一大半字节)。详见 [[gpu-cloud-ssh-background-jobs]]。
2. **下载/整理不吃 GPU**(纯 CPU/IO) → 可租**便宜 CPU 实例**专门下数据，别为下载烧 GPU 钱；或训练紧接就用 GPU 盘。

## 0. 环境坑（py3.8 镜像）

box 是实验1 的 `PyTorch1.11/py3.8/cu113` 镜像，conda 只有 `base`。跑 adapter/build 需要：
- `pip install "pydantic>=2"` —— 契约用 pydantic v2。
- `pip install eval_type_backport` —— **py3.8 关键**：schema 用 `str | None`（PEP604），pydantic v2 在
  py3.8 无法 eval 该注解，装此包即可（报错信息会直接点名它）。
- 跑法：`cd /root/autodl-tmp/ect && PYTHONPATH=src python -m edge_cam.data.adapters.detect.build ...`
  （代码用 scp 上传，非 git clone —— 仓库私有，box 无 gh 鉴权）。
- fiftyone（OIV7 用）：py3.8 需老版本，单独处理，见 §OIV7。

## 1. ENA24 ✅ 映射核验通过

- 文件：`ena24_public.json`（2.9M）+ `ena24.zip`（3.6G 图）。`8789` 图，file_name 扁平数字（`1.jpg`）。
- `audit_unmapped()` 实测：**只有 `Vehicle(138)` / `Horse(63)` 未映射** —— 正是要 DROP 的；其余 21 类全命中。
- 处理：全带框 → `exhaustive=True`；无 location/seq → **按 image 划分**；`negative_quota=0`（~无空图）。
- 下载：`storage.googleapis.com/public-datasets-lila/ena24/{ena24_public.json,ena24.zip}`。

## 2. Caltech CT ✅ 映射核验通过；⚠️ 图源换 ECCV18 下采样

- **107GB 全量图弃用**：`cct_images.tar.gz` 实测 **107GB**（全 244k 图），太大；逐图 HTTP 访问
  `…/cct_images/<uuid>.jpg` 实测 **404**（不暴露单图）。
- **改用 ECCV18 CCT-20 下采样集** `eccv_18_all_images_sm.tar.gz`（**6.2G**，57,864 图，下采样——
  检测 320/416 输入足够）。`eccv_18_annotations.tar.gz`（2.9M）**自带 bbox + location/seq_id + empty(id 30)**，
  与 `caltech_bboxes_20200316.json` 用**同一 UUID 文件名**。
- 文件名核验：bbox json 63,025 图 ∩ ECCV sm 57,864 图 = **45,949 张带框图存在**（其余为别的 location，
  sm 子集不含）。→ adapter 需**按文件存在过滤**（只产已下到的图的记录）。
- `audit_unmapped()` 实测（对 `caltech_bboxes`）：未映射 = `car(2557)/lizard(177)/insect(7)/empty(186)` ——
  car/lizard/insect 正确 DROP；empty 不映射 → 0 框 → 负样本。
- 处理：`exhaustive=True`；**按 location 划分**防泄漏；empty 帧（图级 json / ECCV 的 empty 类无 bbox 注解）
  → 负样本，`negative_quota=12000`。许可 CDLA-Permissive 可商用。
- 待办（代码）：CocoJsonAdapter 加「无 bbox 注解跳过」+「按图存在过滤」；CCT 图源指向 ECCV sm 解压目录。

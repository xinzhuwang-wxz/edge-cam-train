**2. 组装**（加新源只改 yaml）
```bash
# 仓库默认配置：configs/data/detect_feeder5.yaml（raw_root=data/raw/detect，4 源全开）
# box 上用 detect_box.yaml（raw_root=/root/autodl-tmp/detect_raw）：
```

**3. 产物**（落 `processed/`）
- `manifest_{train,test,eval_feasibility}.json` — `DetectionManifest`（带 provenance/split，可加载/溯源）。
- `labels/{train_train,train_val,test_test}.json` — NanoDet 吃的 COCO labels（由 manifest 派生）。
- `license_manifest.csv` — 逐图 path/source/license 署名清册（商用披露）。
- `summary.json` — 各 split 计数。

**4. 喂 NanoDet 训练**：`img_path = manifest.root`(raw_root)、`ann_path = labels/train_train.json`
（val 用 `train_val.json`，测试用 `test_test.json`）；类别 5；详见 [02](02-训练与评估.md)。

**5. OIV7 下载注意**：跑 `fetch_oiv7_direct.py` 前必须 `source /etc/network_turbo`（学术加速），

## 0. 环境坑（py3.8 镜像）

box 是实验1 的 `PyTorch1.11/py3.8/cu113` 镜像，conda 只有 `base`。跑 adapter/build 需要：
- `pip install "pydantic>=2"` —— 契约用 pydantic v2。
- `pip install eval_type_backport` —— **py3.8 关键**：schema 用 `str | None`（PEP604），pydantic v2 在
  py3.8 无法 eval 该注解，装此包即可（报错信息会直接点名它）。
- 跑法：`cd /root/autodl-tmp/ect && PYTHONPATH=src python -m edge_cam.data.adapters.detect.build ...`
  （代码用 scp 上传，非 git clone —— 仓库私有，box 无 gh 鉴权）。
- fiftyone（OIV7 用）：py3.8 需老版本，单独处理，见 §OIV7。

## 1. ENA24 ✅ 映射核验通过

- 文件：`ena24_public.json`（2.9M）+ `ena24.zip`（3.6G 图）。`8789` 图，file_name 扁平数字（`1.jpg`）。
- `audit_unmapped()` 实测：**只有 `Vehicle(138)` / `Horse(63)` 未映射** —— 正是要 DROP 的；其余 21 类全命中。
- 处理：全带框 → `exhaustive=True`；无 location/seq → **按 image 划分**；`negative_quota=0`（~无空图）。
- 下载：`storage.googleapis.com/public-datasets-lila/ena24/{ena24_public.json,ena24.zip}`。

## 2. Caltech CT ✅ 映射核验通过；⚠️ 图源换 ECCV18 下采样

- **107GB 全量图弃用**：`cct_images.tar.gz` 实测 **107GB**（全 244k 图），太大；逐图 HTTP 访问
  `…/cct_images/<uuid>.jpg` 实测 **404**（不暴露单图）。
- **改用 ECCV18 CCT-20 下采样集** `eccv_18_all_images_sm.tar.gz`（**6.2G**，57,864 图，下采样——
  检测 320/416 输入足够）。`eccv_18_annotations.tar.gz`（2.9M）**自带 bbox + location/seq_id + empty(id 30)**，
  与 `caltech_bboxes_20200316.json` 用**同一 UUID 文件名**。
- 文件名核验：bbox json 63,025 图 ∩ ECCV sm 57,864 图 = **45,949 张带框图存在**（其余为别的 location，
  sm 子集不含）。→ adapter 需**按文件存在过滤**（只产已下到的图的记录）。
- `audit_unmapped()` 实测（对 `caltech_bboxes`）：未映射 = `car(2557)/lizard(177)/insect(7)/empty(186)` ——
  car/lizard/insect 正确 DROP；empty 不映射 → 0 框 → 负样本。
- 处理：`exhaustive=True`；**按 location 划分**防泄漏；empty 帧（图级 json / ECCV 的 empty 类无 bbox 注解）
  → 负样本，`negative_quota=12000`。许可 CDLA-Permissive 可商用。
- 待办（代码）：CocoJsonAdapter 加「无 bbox 注解跳过」+「按图存在过滤」；CCT 图源指向 ECCV sm 解压目录。

## 3. COCO 2017（仅可行性）

## 4. Open Images V7 → 弃 fiftyone，改官方直下（关键源：bird/person/平衡）

实操踩坑链：
1. **fiftyone 装不上**：py3.8 装 `fiftyone<0.24` 卡在 `fiftyone-db` wheel 构建（要下内置 mongo，
   从国内挂死）→ **放弃 fiftyone**。
2. **AWS S3 直下被限速**：`open-images-dataset.s3.amazonaws.com` 单图 232K 耗 **22.9s**（~10KB/s）——
   和 cocodataset.org 一样被墙限速。
3. **AutoDL 学术加速救场**：`source /etc/network_turbo`（http_proxy）→ 同图 **2.6s**（~9×）；
   P=16 并行 ~3.6 图/s → 几万图可在数小时拿下（无卡模式便宜）。
4. **annotation 在 googleapis（快）**：`class-descriptions-boxable.csv`、`oidv6-train-annotations-bbox.csv`
   都从 `storage.googleapis.com/openimages` 下，秒级。

**直下方案**（绕开 fiftyone）：bbox CSV 按 MID 过滤 → 选图(每类限额) → 加速并行 S3 下图 →
PIL 读宽高把归一化框转绝对 → 生成 COCO json → `CocoJsonAdapter`（OIV7_LABEL_MAP）。
脚本 `scripts/fetch_oiv7_direct.py`；adapter `oiv7_direct.Oiv7DirectAdapter` 注册为 `open_images_v7`。
MID（核验）：Bird/m/015p6 Squirrel/m/071qp Cat/m/01yrx Person/m/01g317 Dog/m/0bt9lr
Raccoon/m/0dq75 Rabbit/m/06mf6 Fox/m/0306r Deer/m/09kx5 Mouse/m/04rmv Hamster/m/03qrc
Hedgehog/m/0cl4p Skunk/m/0km7z。**only `Mouse` 不要 `Computer mouse`**。
平衡目标（补 CCT 缺口）：bird 多、person 多、squirrel/cat 中，other_animal 少（CCT 已 26k）。

## 4b. /autodl-pub 本地公共数据集镜像（重要发现）

AutoDL 挂了 7TB `/autodl-pub/data`，本地速度。含 **COCO2017**（zip）、**CUB200-2011**、ImageNet、
VOCdevkit… **无 Open Images**。→ **COCO 改从本地镜像解压**（不再走限速的 cocodataset.org）：
`unzip /autodl-pub/data/COCO2017/{annotations_trainval2017,val2017}.zip -d feasibility/coco/`。

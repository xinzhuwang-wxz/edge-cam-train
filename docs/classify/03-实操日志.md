## 2026-06-22 · 三源合并 manifest + Lite0 训练启动（5090）

**数据源（feeder 4src → 实际 3 源；norwegian 弃用）**：`fetch_gbif_birds.py` 按 datasetKey + Aves +
CC0/CC-BY 逐种下图、增量写 index.csv（中断不丢）。最终落库（`/root/autodl-tmp/classify_raw`，43G）：

| 源 | 种 | 图 | license |
|---|---|---|---|
| naturgucker | ~300 | 59023 | CC-BY |
| arter | 200 | 33365 | CC-BY |
| inat | 116（trickle 到稀有种太慢，手动切） | 23141 | CC0/CC-BY（R&D，[[ADR-0005]]） |
| norwegian | — | 弃用（残留 index 损坏 2001 行 vs 27438 图，且源 trickle 慢） | — |

**组装**（`python -m edge_cam.data.adapters.classify.build --config configs/data/classify_feeder.yaml`，CPU）：
- 多源按**学名**合并（`IdentityTaxonomy`；eBird crosswalk 留后续做区域 mask / genus-family aux）→ **360 类**。
- 逐图 license 过滤：**0 丢弃**（fetch 已预过滤 NC）→ CC-BY 109834 + CC0 5695。
- 防泄漏 split（group_key=observer）：train 96128 / val 9545 / test 9856，**每 split 全 360 种**。
- 产物：`processed/manifest.json` + `license_manifest.csv`（逐图署名）+ `summary.json`。
- 坑：① 多源合到一个 root 会路径撞名 → adapter 加 `path_prefix`（record.path=`<源>/images/...`，可移植）。
  ② box 的 `ect` 是代码副本非 git、`edge_cam` 未装 → 用 `PYTHONPATH=src` 跑；`configs/` 子树需补传。

**训练**（5090，cu128，screen `clstrain`，log `/root/autodl-tmp/cls_train_lite0.log`）：
```
PYTHONPATH=src python -m edge_cam.train.classify.train \
  data.manifest=.../processed/manifest.json data.data_root=/root/autodl-tmp/classify_raw \
  data.balanced_sampler=true data.batch_size=128 data.num_workers=16 \
  trainer.accelerator=gpu trainer.max_epochs=80 \
  model.name=efficientnet_lite0 model.pretrained=true \
  output_dir=outputs/classify/feeder_lite0_224 export.enabled=true export.opset=13 hydra.job.chdir=False
```
- `fast_dev_run` 烟测先过（pretrained 加载经 hf-mirror、GPU 前后向、balanced_sampler 均 OK，train_loss≈ln360）。
- 待训完：val/test top-1/top-5（overall + macro）、INT8 envelope（ORT-QDQ）、区域 with/without、导 FP32 ONNX。

## 二、三个被反复印证的核心模式（多路独立佐证）

### 模式 1｜地理先验 = 独立分布模型 × 视觉分数的**晚融合乘法、推理期、不进模型**
- `P(种|图,位置,月) ∝ P(图|种) · P(种|位置,月)`。iNat 实测 **+6.6~9%**。
- **可 OTA 独立更新先验、不动 INT8 主模型**（换区域清单不重训图像模型）——晚融合的工程优势正是端云+OTA 要的解耦。
- **坑照抄**：地域清单外的种给**小先验值 k 而非清零**（iNat 单此 +2.4%），避免层级回退/未覆盖种被硬杀。
- 与硬规则契合：先验留 A7 CPU 后处理，不进 NPU 图。

### 模式 2｜层级回退 = **沿分类树分数上滚 + 置信门**（非简单阈值拒识）
- iNat"共同祖先 roll-up"（种分数上滚找高置信属/科节点）+ Seek 7 级 ladder 是成熟产品形态。
- 比单阈值稳，对长尾稀有种尤其。**把 plan-v2 已定的层级回退从纯后处理，配合训练期 genus/family aux loss（见 04），预测本身更准。**

### 模式 3｜分类头 →**embedding 检索**的可扩展演进路（四合一杠杆）
- PlantNet 证明 embedding+kNN 在细粒度上 ≥ softmax（+10pt，类越多优势越大）。
- 一次解决 **长尾 + 新增种不重训 + 开放集拒识 + OTA 成本** 四个问题：新增种=往云端参考库加 embedding，零重训；最近邻超阈值→天然 open-set 拒识→回退属/科；OTA 只推库不推 .nb。
- **对上** plan-v2"低置信/不在地域清单/稀有种→留云锚点"设计意图。
- ⚠️ **两条落地风险**（务必进 gate 包络）：① **embedding 对 INT8 量化比分类模型敏感得多**（人脸工程经验：无脑 INT8 召回会崩），Vivante INT8-only 上是最大风险，须专列"embedding INT8 cosine 检索召回"评估、可能要输出层保精度/精细 per-channel 校准；② 需**现在就在 ONNX 导出保留 penultimate FP32 embedding 出口**（事后补很贵）。

## 三、teacher 困境的解法（重要更新 ADR-0005 C7 的现实）

之前结论：外部鸟类视觉 teacher 全因 NC 传染判红（BioCLIP MIT 权重但 TreeOfLife 混 CC-BY-NC）、SpeciesNet 粗粒度、Perch 音频、Merlin 闭源 → 只能自训 clean teacher。**新发现给了合规的第二条路：**

- **DINOv2（Meta，已重授权 Apache 2.0）** 冻结主干 linear probe：iNat2021 85.7% / CUB 91.6%。**把它当云端强特征器 + LIFT 式轻 head（冻结主干只训 ~0.18M 参数，反而保护尾类）→ 蒸馏进 EfficientNet-Lite 端侧 student**，全程上游 FP32 ONNX、不碰端侧 INT8、**许可全干净**。有现成开源蒸馏实现（Scale-KD to ConvNeXt/MobileNet/EfficientNet）。
- **DINOv3（2025.08）官方发布纯 ConvNeXt CNN**（从 ViT-7B 异构蒸馏，iNat2021 ~87%）——架构上极适合"端侧只能 CNN"，**但自定义"DINOv3 License"非 Apache/MIT，商用海外须法务审查**，别默认可商用。
- **仍判红**：BioCLIP/BioCLIP-2（权重 MIT 但训练含 CC-BY-NC）、BirdNET（CC-BY-NC-SA）、Xeno-canto（混杂 NC）。
- **结论**：ADR-0005 C7"第一版不蒸馏 NC teacher"仍对，但**可补充**："用 Apache 的 DINOv2 当特征/teacher 蒸到 Lite student"是合规增量，建议开 ADR 记录。另有 BirdCLEF 验证的 **Noisy-Student 自蒸馏**（自训集成当 teacher）作纯干净备选。

### 3.0 先问：要不要 teacher / 蒸馏？（**不是必需**）
- **baseline 根本不需要 teacher**：直接用 timm ImageNet 预训练的 Lite4 + 你的 CC0/CC-BY 鸟种数据 fine-tune 就能用（上一轮 Lite0 val 0.748 就是这么直接训的、没 teacher）。**teacher/KD 是可选提分杠杆（通常几个点），不是地基**。→ 正确节奏：先直接训 student 拿 baseline，teacher 作后续消融的一列，别一开始就上。
- **直接部署 teacher？不行**：DINOv2 是 ViT（attention/LayerNorm 在 Vivante 软回退→崩），最小 ViT-S 也 21M FP32；DINOv3 有 ConvNeXt 版但最小 29M + 自定义许可。teacher 天生不能端侧部署——这正是要一个原生小 CNN student 的原因。
- **剪枝 teacher？不适配这个平台**：剪 ViT 到能在 Vivante 跑 = 等于拆掉 attention 换架构，不现实；剪 CNN teacher 到 Lite 级通常不如"直接训一个 Lite + 蒸馏"。剪枝不是这条工具链的好路子。
- **"小 DINOv2 能端侧用吗"——概念分层**：DINOv2 有 ViT-S/B（从 ViT-g 蒸馏的小版本）**但仍是 ViT，只能云端当 teacher/特征器**。端侧"小 CNN 预训练"来源是 **timm 的 ImageNet 预训练 Lite/RepViT/MNV4**（你已在用 `pretrained=true`），或 DINOv3-ConvNeXt（许可待法务）。"小"≠"端侧能用"，**架构（ViT vs CNN）才是关键**。
- **"用大模型打标就不用 teacher 了？"——是的，基本可以**：大模型打伪标（pseudo-label）= **数据层蒸馏**，比在线 KD 工程更轻（一次打标 vs 训练时在线跑 teacher），这就是 **Noisy-Student** 思路。两个约束：① **打标模型同样不能 NC**（用 NC 模型打标 → 伪标带 NC 传染，同 §3.2 边界）；② **伪标要 QC**（细粒度伪标有噪声 → Cleanlab 洗 + 置信度过滤）。**判断：打标扩数据 + 直接训 student，优先于在线 KD**；在线 DKD 作"打标后还想再榨几个点"的叠加项。

**提分手段的推荐上手顺序**：直接训 baseline → 大模型（DINOv2/自训）干净打标扩数据(Noisy-Student) + Cleanlab QC → 层级 aux loss → 在线 DKD → geo 先验重排。别一步到位堆蒸馏。

### 3.1 先分清三个角色（别混）
| 角色 | 是什么 | 约束 | 选谁 |
|---|---|---|---|
| **端侧 student backbone** | 真正部署在 V861 上跑的分类器 | INT8 / 纯 CNN / 内存克制 | EfficientNet-Lite4 / RepViT-M1.5 / MNV4-Conv-M（见 04） |
| **云端 teacher / 基模型** | 训练时提升 student、**不部署** | **不受端侧约束**（云端 FP32，可很大） | DINOv2(Apache) 冻结特征 / 自训集成 |
| **云端检索参考库 encoder** | 出 species embedding 建 kNN 兜底库 | 与 teacher 统一同一 encoder 才可比对 | = 同上 DINOv2（与 KB embedding 表对齐） |

关键：**teacher 在云端、不受 V861 INT8/纯CNN 约束**——这解放了选择，teacher 可以是 ViT-g 级大模型，只要最终蒸出的 student 是端侧 CNN、上游只产 FP32 ONNX。

### 3.2 NC 传染的精确边界（为什么 DINOv2 能用、BioCLIP 不能）
判红看**训练数据**、不看权重 license 字面：
- **DINOv2**：通用自监督，预训练数据 = Meta 自建 LVD-142M（**非 NC 鸟类数据**），权重 **Apache** → 当特征/teacher 不触发 NC 传染。蒸出的 student 许可 = Apache(DINOv2) + 你蒸馏所用鸟类数据(须 CC0/CC-BY) = **干净可商用**。
- **BioCLIP**：权重虽 MIT，但**本身就是用 CC-BY-NC 鸟类数据训的** → 蒸它 = 间接吸收 NC 鸟类知识 → student 传染，**判红**（ADR-0005 §4"换 head 不洗白"）。
- 这条边界同样适用 KB（[[04]] 协同）：KB embedding 若用 BioCLIP，商用侧不能沿用，须换 DINOv2。

### 3.3 适配我们场景的具体路径（推荐主路）
`DINOv2(Apache, 云端冻结)` → **LIFT 式只训 ~0.18M 轻 head**（冻结主干、保护尾类，iNat2018 79–80%）在**你的 CC0/CC-BY 鸟类数据**上 → 得到 **clean 鸟类 teacher** → **Decoupled-KD**（拆 target/non-target，细粒度友好）+ 低分辨率 DKD（对症 crop 小目标）**蒸到端侧 Lite student** → 上游只导 **FP32 ONNX** → 板端 ACUITY/pegasus PTQ → `.nb`。全链路许可干净、不碰端侧约束。
- **三条路线权衡**：**A. DINOv2 蒸馏**（主路，许可干净、证据强、有现成 Scale-KD 实现）；**B. 自训集成 + Noisy-Student 自蒸馏**（最纯净但要自己训大模型，算力贵）；**C. DINOv3 纯 ConvNeXt**（架构最贴端侧，但自定义许可须法务，暂缓）。→ 主推 A，B 作纯净备选，C 待法务。
- **teacher encoder 一处两用**：同一 DINOv2 既当蒸馏 teacher、又当云端检索参考库 encoder，天然与 KB embedding 表对齐（[[04]]）。

## 四、时空出现频率先验（回应"特定鸟×特定地区×每月频率"诉求 + 许可）

- **⚠️ 月份 ≠ 独立系统**：月份和地理是**同一个时空先验 `P(种 | 经度, 纬度, 一年中第几天)` 的不同输入维**，不是两套东西。geo_prior 就是一个函数同时吃三维。别把"地理先验"和"月份先验"当两个模块建。
- **eBird Status & Trends**（周/季丰度标杆，背后 = **AdaSTEM** 时空模型，输出每种×每周×~3km 相对丰度；那些"某鸟全年每周出现频率"动画/曲线多半就是它，迁徙流动动画则是基于它的 **BirdFlow**）→ 但 Cornell 专有商用条款、**商用禁用、无 CC 子集**（ADR-0005 C6 判得对）；BirdFlow 基于 S&T 数据同样受限。**产品/权重拿不了，但能力可自建（下）**。
- **但好消息**：eBird 出现记录经 **GBIF EOD 是 CC-BY 4.0、可商用**——合法拿 eBird 数据的入口，不必完全放弃 eBird。
- **最该复用的架构蓝本 = Mac Aodha `geo_prior`（MIT 代码，原生带 day-of-year 月份维度）**：`P(物种|经纬度,一年中第几天)`，presence-only 训练、显式建模拍摄者偏差、专为"融进图像分类"设计。比 base SINR 好（SINR 无时间维）。
- **许可干净自建路线**：复用 geo_prior 架构 + **GBIF CC0/CC-BY occurrence（含 eBird EOD CC-BY）自训**"物种×地区×月份"软先验；产物几 MB、可端侧携带、推理期查表。**绝不碰 S&T 丰度、绝不用官方预训练 SINR/geo_prior 权重（provenance 掺 NC）、绝不掺 CC-BY-NC**。稀有种观测不足（<50–100 条）→ 回退属/科级先验（与层级门控对齐）。
- **融法**：(a) **推理期软权重重排**（现方案低成本升级，强推先做，留 A7 CPU）；(b) 时空特征条件输入进模型（增益略高但要重训、破坏 OTA 解耦，仅作后续消融）。

## 五、超多类 / 长尾工业方法

- **类数到"数千"普通 softmax 还扛得住**；真正逼你换架构的是**长尾 + 频繁加新种**→ embedding/检索式主场（见模式 3）。extreme classification 树模型（PECOS 等）是给十万+标签准备的，暂用不上。
- **长尾最稳组合（端侧零/近零推理开销）**：instance-balanced 学表征 + **logit adjustment / balanced softmax**（推理期减 log-prior，零成本）+ **class-balanced loss** + **cRT/LWS 或 LOS(ICLR2025，免类分布先验)** 重训分类头。花哨重采样/GAN 对端侧不划算。
- **区域先验可作为 logit adjustment 在推理期叠加**（用先验频率替训练频率），不重训——与模式 1 天然合并。
- **model-level MoE**（按地域切换轻量专家/参考库子集，非网络内路由）端侧可行；网络内 MoE 出局（内存+Vivante 算子）。

## 六、data-centric：ROI 高于换 backbone（2024-2026 一致结论）

- **标签清洗 > 加数据**：Cleanlab / Confident Learning（**Apache 2.0**）多领域 +10~74%；**iNat research-grade 非正式估 ~15% 标签错**，稀有类 10% 错标可能精度腰斩。
- **半监督+伪标 QC（细粒度专用）**：标准 FixMatch 的 strong-aug 毁羽色纹理；**PEPL**（CUB 20% 标注 +13%）、soft-label 是细粒度正解；GBIF research-grade 当弱标注池。
- **合成数据（有条件）**：DA-Fusion 对细粒度 +24% **但必须带背景上下文**（"栖在树枝的鸟"才有效，孤立鸟无效），极稀有地域种慎用。
- **对本项目**：Lite0 级小模型上，"洗 iNat 标签 + 换 LA loss + 解耦重训 head"的收益很可能**超过 Lite0→Lite4 的架构升级**，应作消融矩阵**前置独立列**。全部许可干净（Cleanlab Apache + 已有 CC0/CC-BY 数据）。

## 七、评测口径升级（影响 gate 与命门指标定义）

**top-1 已被 2024-2026 学界明确判"不够用"**，产品级"好用"应多尺子并读：
1. **层级可用率**（种/属/科，模型自选预测深度）——与门控逻辑一一对应；纯种级评测脆（H-CAST 84.9→45.1）。
2. **校准（门控的地基）**：稀有种普遍**过自信**→"高置信才出片上标签"在长尾尾端悄悄出错。补 **per-class Platt Scaling（>全局 temperature）+ Reliability Diagram + ECE（长尾用 Esbin-ECE）**。
3. **拒识 = risk-coverage / AURC**：产品指标用 **Coverage@Precision=0.9** 和 Precision@Coverage；**物种特定阈值 > 全局阈值**（BirdNET 2025 产品化验证）。
4. **区域内 vs 全局双精度**（如 iNat 分报"CV"与"CV+Geo"两数）。
→ 现 gate（fp32/int8 四维阈值）应扩这几个口径；**物种特定阈值**替代全局是低成本高回报改动。

## 八、部署后自适应（端侧 INT8 纯前向的现实）

| 手段 | 端侧可行 | 说明 |
|---|---|---|
| 地理先验 logit 调整 | ✅ | A7 CPU 浮点，几百 μs，不碰 NPU |
| per-location 置信阈值 | ✅ | 标量参数 |
| 原型库/embedding 检索更新 | ✅ | NPU 出 embedding，A7 更新类中心/kNN，无梯度 |
| AdaBN / 梯度型 TTA(TENT/EATA) | ❌ | Vivante 折叠 BN 进 INT8 权重、无反向传播 |
| 联邦学习 | ❌ | 需 FP32 算力 |

→ 端侧个性化押 **地理 logit + per-location 阈值 + 原型库更新**（相机 GPS 固定 = 免训练个性化，把万类压到 50–200 类）；真正的学习闭环放**云端人在环飞轮**（用户确认→research-grade→每 1–2 月云端重训/蒸馏→OTA 下发 .nb+参考库，HITL 把误差 34.9→8.7%）。

## 九、整体架构蓝图（若今天从头设计）

**端侧（V861，INT8 纯前向 CNN）**
- 检测（NanoDet-Plus 固定，后处理留 A7）→ bird crop
- 分类 backbone = EfficientNet-Lite 级 CNN（可由云端 DINOv2-Apache 冻结特征 + LIFT 轻 head 蒸馏而来）
- **双出口**：(a) softmax logits 本地快判；(b) **penultimate FP32 embedding 出口**供云端检索/原型更新（**现在就设计进 ONNX**）
- 本地：地理先验 logit 调整（GPS+月份→清单压候选到 50–200 类）+ per-species/per-location 阈值门控 + MSP/energy 拒识
- 门控三态：高置信→片上种级；中置信→层级回退属/科；低置信/OOD/不在清单→留云锚点

**云端（可训练，FP32）**
- embedding 参考库 + FAISS kNN（新增/稀有/长尾=加参考 embedding，零重训；OTA 只推库不推 .nb）
- geo_prior/SINR 重排（自训、许可干净）
- 人在环飞轮：确认→research-grade→定期重训/蒸馏→OTA
- 可选音频第二通道（Perch 路线，须先清许可；不进 v1 主线）

**地理时间融合**：独立 geo 模型 + 推理期后验乘法（别注入视觉主干，保 OTA 解耦）。

## 十、对 roadmap / 命门的含义 + 新增决策

**最该重视、可能被忽略的 3 件事**：
1. **分类头 → embedding + 云端检索** 一举解长尾/新增不重训/拒识/OTA 四问——**代价是现在就要在 ONNX 保留 FP32 embedding 出口**（事后补贵）。这是最被低估的架构杠杆，但 embedding 的 INT8 量化风险须先验证。
2. **校准是门控地基**——补 per-class Platt + Reliability + 物种特定阈值，成本低回报直接。
3. **data-centric ROI > 换 backbone**——洗标 + LA loss + 解耦重训，作消融矩阵前置列。

**命门指标的含义**：调研强烈指向"命门 ≠ 单纯 top-1"，而是**层级回退后的"可用率"@校准可靠 + 区域内精度**（报对种 or 安全回退属/科算可用，惩罚自信报错种）——与 iNat/Seek 产品、层级 aux、校准评测三者同构。

### 姊妹 repo 协同：`bird-tagger/taxonomy`（现成地基）
用户另一 session 在搭 `/Users/bamboo/Githubs/bird-tagger/taxonomy/`——**dataset-independent eBird 物种 registry + 多模态 KB**，对本项目是现成地基：
- **规模答案**：全 eBird/Clements 2025.0（sha256 版本锁）= **11,167 species / 45 orders / 251 families / 2,377 genera**，rollup 4,120。→ **11,167 = 云端兜底天花板；端侧装区域裁剪集（几百~两千，与 Merlin/iNat 一致）**。本项目现 360 类只是起点。
- **直接复用**：① `species.jsonl`(ebird_code 主键 + genus/family/order) = **ADR-0002 taxonomy seam 的现成实现**，可填 IdentityTaxonomy 占位债；② `rollup` + 层级列 = **层级回退"分数上滚"树骨架现成**（模式 2）；③ `avibase_id`(78.7%) = 比纯 eBird crosswalk 更鲁棒的**跨源合并概念锚**（三源合并可升级用它）；④ KB 的 **trait 表**（受控词表属性→近似种区分/回退解释）+ **embedding 表**（#14 stub→**云端 kNN 检索参考库骨架**，与模式 3 embedding 路线呼应）。
- **协同锚点** = `ebird_code` + `taxonomy_version`(raw_sha256 pin)。两 session 应在此对齐；embedding 接口互为参考库。工程纪律（版本 pin/provenance/AI 不覆盖/版本化 embedding）两 repo 一致。

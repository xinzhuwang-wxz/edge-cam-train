# 01 · 细分类数据集方案

> 目标：用**可商用**开源 + (未来)自建数据，训出端侧鸟种细分类器。本文定：类别体系、数据源盘点、
> license 分层、iNat/GBIF 下载清洗、taxonomy crosswalk、区域候选集、防泄漏切分、数据量目标。
> 决策见 [[ADR-0005]]（数据许可红线 + teacher + 区域）、[[ADR-0001]]（可行性优先）、[[ADR-0002]]（eBird taxonomy）。
> **许可从严**（海外发行）：训练只用 CC0/CC-BY + 自建，逐图过滤，传染规则照 §4。

## 1. 类别体系（稳定物种表 + 层级输出）

**不要用英文鸟名当类 ID**，建稳定物种表（版本化数据产物，[[ADR-0002]]）：
```
taxon_id（内部稳定 ID）· scientific_name · genus · family
ebird_code（产品对外稳定 ID）· inat_taxon_id · gbif_taxon_key（合并 hub）
common_name_en · common_name_localized · region · edge_enabled
```
- **层级输出**：species → genus → family → bird。无把握**不强猜种**，回退 genus/family/unknown。
- eBird 物种**代码**当 ID 串可用（受限的是**频率数据**，见 §7/[[ADR-0005]]）。
- 实验1：birds525 用英文俗名、408/525 映射到 eBird——本方案以物种表替代，全量映射。

## 2. 数据源盘点（实测，[[ADR-0005]]）

> Aves = 鸟纲（所有鸟），GBIF `taxonKey=212`。数字来自 GBIF API 实测（occurrence-license proxy，
> 真正图片用前还要逐条 `media.license` 复核，会更少）。**两条线、两个池**：图像（要图）vs 区域先验（只要 occurrence）。

**① 训练图像**（要图 + CC0/CC-BY，按种限额取子集，**不下全量**）

| 源 | 可商用鸟图（CC0/CC-BY） | 类型 | 角色 / 许可 |
|---|---|---|---|
| **iNaturalist Open Data** | ~482 万 | 公民科学实拍，全球 | **先用（R&D）**，主力；北美/规模靠它。⚠️ iNat **ToS 禁商用 AI 训练（连 CC0）**+ 元数据表无 license → **商用发行前必须解决许可**（书面澄清 or 替换），见 [[ADR-0005]] |
| **NABU naturgucker**（德） | 671k | 公民科学实拍 | 欧洲主力，CC-BY |
| **Norwegian Species Obs**（挪） | 572k | 公民科学实拍 | 欧洲，CC-BY |
| **Arter.dk**（丹） | 138k | 公民科学实拍 | 欧洲，CC-BY |
| **自建喂鸟器**（未来） | — | 真实 feeder crop（遮挡/背身/夜视…） | 决定真实效果，自有可商用 |

> 池子总览：Aves 有图 42.6M（任何许可）→ CC0/CC-BY 仅 ~737 万（~16%，余 ~84% 是 NC 不用）→ 其中 iNat ~482 万、
> 非 iNat ~254 万。非 iNat 里**可用实拍**≈ naturgucker+挪+丹 ~1.4M（**偏欧洲**）；**博物馆标本**（NMNH/Naturalis/NHM，
> CC0 但死鸟剥制）**域不符、排除**。→ 没有 iNat 则北美薄、靠 feeder。

**② 区域 / 月份先验**（只要 occurrence：种 + 经纬 + 日期，**不要图**，见 §7）

| 源 | 量 | 角色 |
|---|---|---|
| GBIF Aves occurrence CC0/CC-BY（含坐标） | **~21 亿**（CC-BY 19.8 亿，**含 eBird 原始观测 EOD=CC-BY**；CC0 1.4 亿） | 聚合 `f(种,区域,月)` 先验表（不存原始，§7/§9） |

**③ 物种 ID / taxonomy**：eBird/Clements checklist（代码=稳定 ID、学名、科）+ crosswalk（见 §6）。

**排除**：NABirds/CUB（非商用、含评测）· eBird **Status&Trends 丰度**（商用禁；但 eBird **原始观测**在 GBIF 是 CC-BY，
可用于先验）· BioCLIP/BioCLIP2（训练数据含 NC，污染，不当 teacher）· SpeciesNet（Apache 但服务器级 EfficientNetV2-M
~54M + 输出授权灰区，第一版不用）· 博物馆标本（域不符）。

## 3. license 分层（决定进训练还是排除）

| 层 | 源 | 用途 |
|---|---|---|
| **commercial-clean** | naturgucker/挪/丹（CC-BY）· GBIF 非 iNat CC0/CC-BY · 自建 feeder | **进训练**，权重无传染、可发行 |
| **R&D-先用**（商用前需澄清） | **iNaturalist**（逐图 CC0/CC-BY + research + Aves + species 过滤） | **开发/实验用**；上市前必解决 iNat 许可（[[ADR-0005]]，标 open-item） |
| **prior-only** | GBIF occurrence CC0/CC-BY（含 eBird EOD CC-BY） | 仅建区域/月先验（不要图） |
| **excluded** | NABirds · CUB · eBird 丰度 · BioCLIP/含 NC teacher · 博物馆标本 | 不碰（NABirds/CUB 连评测都不用） |

- 逐图 `attribution`（作者/login/URL/license）→ 署名清册随产物披露（CC-BY 要求；GBIF 引 DOI）。
- 传染：teacher 含 NC → student 不可商用（§4）→ 第一版不蒸馏。
- **iNat 决策**：先用着把模型/流程做出来，**真商用再澄清/替换**（[[ADR-0005]] 决策 1）。

## 3b. DatasetAdapter 抽象（每数据集自包含，复用检测模式，[[ADR-0003]]）

和 `adapters/detect/` 同构 → 新建 `src/edge_cam/data/adapters/classify/`：每源一个**自注册 adapter**，
声明 spec、实现 `load_raw()`；基类收口公共逻辑。**加新源 = 写一个 adapter，组装/训练 caller 不改。**

```
ClassifyDatasetAdapter(基类)：
  spec: name / license_allow({CC0,CC-BY}) / field_map / taxonomy / role(train|eval_only) / clean / split_unit
  load_raw() → 原始样本：path · raw_taxon · license · lat/long · observed_at · observation_id · observer_id
  基类 build_records()：逐图 license 过滤 → taxonomy 映射→taxon_key([[ADR-0002]]) → 丢未映射 →
    去重(sha256+pHash) → 写 provenance(source/license/taxon_key/坐标/日期) → 防泄漏 split(§8)
  assemble(adapters) → DatasetManifest（与分类训练/评估共用，SampleRecord 已带 Provenanced 字段）

具体 adapter：
  InatAdapter   读 iNat 元数据 CSV → 过滤 CC0/CC-BY+research+Aves+species → 建 S3 URL 下 medium
  GbifAdapter   GBIF media 逐图过滤 + 对 iNat 去重(datasetKey)；另路：occurrence → 区域×月先验表(§7)
  FeederAdapter （未来）自建 crop
```
与检测 adapter 的差异（分类特有）：① **license 逐图过滤是一等步骤**（iNat 默认 NC）② 多带**经纬度/日期**
（给区域先验 + 防泄漏 split）③ **区域先验表是 GBIF occurrence 的独立产物**（非图）。
现有 `data/prep.py`（DataPrepConfig 多源 + Taxonomy seam）是前身 → adapters/classify 升级为自包含适配器
（如检测 adapter 取代旧 detection_ingest）。代码落地时机见正文「框架补齐时机」。

## 4. iNaturalist 下载与清洗（主力，annotation-first）

- **桶**：`s3://inaturalist-open-data`（us-east-1，**匿名 `--no-sign-request`**，非 requester-pays）。
  图片 URL：`https://inaturalist-open-data.s3.amazonaws.com/photos/{photo_id}/medium.{extension}`
  （尺寸：original 2048 / large 1024 / **medium 500（训练用）** / small 240；`extension`=jpg/png）。
- **元数据**（月度，gzip TSV）：`photos.csv.gz` `observations.csv.gz` `taxa.csv.gz` `observers.csv.gz`。
  **精确列名（核验）**：
  - `observations`：`observation_uuid`（UUID）· `observer_id` · `latitude/longitude` · `positional_accuracy`
    · `taxon_id` · `quality_grade` · `observed_on`（**无 license 列**）。
  - `photos`：`photo_uuid` · `photo_id` · `observation_uuid` · `observer_id` · `extension` · **`license`（自由文本）** · `width/height`。
  - `taxa`：`taxon_id` · `ancestry`（物化路径）· `rank` · `name`。`observers`：`observer_id` · `login` · `name`。
- **过滤链**（annotation-first：先元数据筛 → 只下需要的图）：
  1. **商用 license**：`photos.license ∈ {CC0, CC-BY}`（逐图，自由文本须先 distinct 校验）。
  2. **Research Grade**：`observations.quality_grade = 'research'`。
  3. **Aves**：join `taxa`，`ancestry` 含 Aves taxon（实测确认 id）；**species 级** `rank='species'`。
  4. 粗经纬有值、非圈养（captive 标记）、去重（sha256 + pHash）。
- **逐图清单**（随产物）：`photo_id/observation_uuid/source/source_url/license/creator(login)/inat_taxon_id/`
  `scientific_name/latitude_coarse/longitude_coarse/observed_at/image_path/sha256/split`。
- ⚠️ 两项需你直接确认（联网查不到）：iNat **元数据本身的 license**（邮件 help@inaturalist.org）；
  **CC0+CC-BY 在 Aves 子集的确切占比**（从 `photos.csv.gz` 自己数）。

## 5. GBIF 补充（长尾/欧洲 + 区域先验源）

- **media**：`media[].{license, creator, identifier, format}`——occurrence 与 media license **独立**，
  occurrence-search `license=CC0_1_0,CC_BY_4_0` 只筛 occurrence 级，**每个 `media.license` 必须客户端再过滤**
  （常是 CC URL 串，归一化；空/非 CC → 不可商用丢）。
- **去重 vs iNat**：iNat Research-grade 数据集 `datasetKey = 50c9509d-22c7-4a22-a47d-8c48425ef4a7`；
  按 iNat observation id 去重（不是 CDN URL）。

## 6. taxonomy crosswalk（无单一三方对照，自建）

- 产品对外稳定 ID = **eBird code**；合并 hub = **GBIF accepted usageKey**；join key = 归一化 scientific_name。
- 构建：eBird/Clements checklist 种子（ebird_code + sciName + family）→ GBIF `/species/match` 取 usageKey
  → iNat `/v1/taxa` 取 inat_taxon_id（Wikidata 交叉校验，它一条 item 同时有 GBIF/iNat/eBird ID）→ Avibase 调和 split/lump。
- **所有 taxonomy 版本 pin 进数据契约**（[[ADR-0002]]）；落地 `EbirdTaxonomy(mapping_file)`（现 `IdentityTaxonomy` 占位）。
- **eBird map 去留**：eBird **物种代码/taxonomy（Clements checklist）仍用**（当 `taxon_key`/物种表稳定 ID，[[ADR-0002]]）；
  实验1 那张 birds525 专用 `ebird_map.csv`（英文俗名→code）**退役** → 改用本 crosswalk（学名/iNat id → eBird code）。
  受限只是 eBird **丰度数据**，代码/分类不受限。

## 7. 区域 + 月份先验（候选集 + 重排，GBIF 自建非 eBird）

设备推理时知道**在哪**（经纬→网格 cell）+ **何时**（日期→月）。鸟的出现随区域+季节变（迁徙）→
`p(species | cell, month)` 是强信息。eBird **丰度产品**商用禁止 → **从 GBIF CC0/CC-BY occurrence 自建**
（含 eBird **原始观测 EOD=CC-BY**，可用）。

- **数据池**：GBIF Aves 带坐标 occurrence **~21 亿 CC0/CC-BY**（实测；CC-BY 19.8 亿大头是 eBird EOD）。
  只要"种+经纬+日期"、**不要图** → 池子远大于训练图（§2 ①）。
- **不存原始 21 亿**（§9）：要么按**目标种 × 目标区域 × 月**查 GBIF 聚合计数（有界），要么 SINR 式在采样子集
  （几百万够）训小空间模型。**输入 21 亿 → 输出几 MB**（稀疏表 / 小网络）。
- **离线建表**：聚合 `f(species, cell, month)`。cell 先粗（国家/州，稀疏 cell 退化到邻格/更粗区域平滑）；
  month 12 桶（迁徙细可 week=52）。
- **端上重排**（软先验，优于硬 mask）：`后验 ∝ 分类器概率 × prior(species|cell,month)`，即
  `logit + λ·log f`。**λ 调先验强度**：λ=0 不加先验（基线）、λ→∞ 等价硬 mask（会误杀迷鸟）。
  软先验给视觉确信的种留翻盘空间，不做地理隧道视野（喂鸟器最珍贵的恰是罕见来访）。
- **为何加月**：迁徙——同片区 5 月有的莺 12 月没有 → 月把季节更替圈进来，候选收得更紧更准。
- **OTA**：`f` 表稀疏存、版本化随 bundle 下发；换地区/季节只换表、不重训不重发模型（[[ADR-0003]] C5）。
- **评估必报**：with/without（λ=0 vs 调优 λ）+ 不加区域/日期的全局基线（量化增益；实验1 区域 mask +1.3pt，
  加月对迁徙种应更多）。
- **落框架**：`eval/regional.py` 的 `RegionalMask`（现硬 mask by taxon_key）→ 扩 `f(species,cell,month)` 软先验。

## 8. 防泄漏切分（关键）

- **禁止按图随机切**：同一 observation 的连拍/同一拍摄者/同一地点高度相关 → 整组进同一 split。
  组键：`observation_uuid` + `observer_id` + location + time window。比例 **train 80 / val 10 / test 10**。
- **测试集分层**：
  - **自有 test split**（主指标，同分布留出）。
  - **自建跨区域/跨源 holdout**（泛化信号，替代 NABirds/CUB 的作用、且合规）。
  - 近似种困难集 / 低质量·unknown 集 / INT8 端侧集 / (未来)feeder test。
- pHash + sha256 去重，尤其 **iNat↔GBIF 跨源重复**。

## 9. 数据量目标（每区域包）

| 类别 | 目标 |
|---|---|
| 高频物种 | 300–1000 张/种 |
| 中频物种 | 100–500 张/种 |
| <50 张/种 | 暂不上端，`edge_enabled=False` → cloud_only |
| 每区域总量 | 10 万–30 万张 |
| 自建 feeder（未来） | 首期 2 万–5 万人工复核 crop |

> 量不是唯一：同一 observation 十张连拍 ≪ 十个不同地点/季节/个体。优先多样性。
> 落地顺序：①写 iNat/GBIF adapter（复用 DatasetAdapter 模式 + DataPrepConfig 多源）→ ②AutoDL 过滤下载 → ③组装 manifest → 训练。

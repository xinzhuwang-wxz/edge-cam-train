# 02 · 细分类训练与评估

> 接 [01-数据集.md](01-数据集.md)。本文定：backbone、区域策略、训练阶段、类不均衡、层级输出+置信门控、
> 评估口径、端云部署、量化。原则：**端侧只认常见种、长尾交云、INT8 算子安全、bird 召回链路上不漏种**。
> 决策见 [[ADR-0005]]（数据/teacher/区域）、[[ADR-0001]]（可行性优先）、[[ADR-0003]]（模型族 seam）。

## 1. backbone：EfficientNet-Lite0（实验1 实测定）

| 模型 | fp32 top-1 | **INT8 掉点** | 结论 |
|---|---|---|---|
| **efficientnet_lite0@224** | 0.921 | **0.19pt** | ✅ **端侧选它**（INT8 后反超） |
| mobilenetv3_large_100@224 | 0.933 | **3.71pt** | ❌ INT8 掉点 ~20× |

- 机理：MobileNetV3 的 **SE + h-swish** 在 INT8 NPU 上掉点大/回退软算子；EfficientNet-Lite **去掉 SE、swish 换 ReLU6**，
  专为 INT8 PTQ 设计。V85x INT8-only → **shipped 模型是 INT8 这个数 → lite0 是赢家**（研究 + 实验1 一致）。
- 扩容备选：Lite1（240px）/ Lite2（260px）。**避**：FastViT/MobileViT（attention）、vanilla RepVGG（INT8 掉点>20%，要 QARepVGG）。
- 输入 224（细粒度需细节：喙/眼/羽）；192 做消融（实验1 跑过）。timm + Lightning + Hydra（[[ADR-0003]]）。

## 2. 区域策略：一个 pipeline 两用（默认全局头 + 推理 mask）

[[ADR-0001]] + 端侧 head 大小约束 → **同一训练 pipeline 支持两种产出，config 切换**：
- **默认**：训**全局头**（所有 edge_enabled 物种）→ **推理期区域 mask**（GBIF 候选清单，OTA 换、不重训，
  实验1 +1.3pt）。运维最简（一个模型 + 多区域 mask）。
- **退化**：当某区域物种太多致 head 超端侧预算/精度不够 → 同一 pipeline 用 **region-filtered 训练集** 训该区域
  **专用模型**（小 head）。即"注册一次、config 切区域"。
- 评估全局头时**必报 with/without mask**（量化区域增益）+ 不加区域过滤的基线。

## 3. 训练阶段（预训练微调，非从零）

| Stage | 做什么 | 数据 | 备注 |
|---|---|---|---|
| **1 通用鸟类** | ImageNet 预训练 → 鸟类通用微调 | iNat+GBIF 合规并集 | species 主 + **genus/family 辅助 loss** + label smoothing |
| **2 区域** | 区域微调 / 或仅推理 mask（见 §2） | 区域子集（GBIF 先验定候选） | 地理约束常比换大模型更有效 |
| **3 喂鸟器** | 加自有 feeder crop，模拟检测器真实输出 | 自建 feeder（未来） | 框抖动/不完整裁剪/多鸟/压缩/模糊/逆光/夜视 |
| **4 蒸馏+量化** | **第一版跳过蒸馏**（[[ADR-0005]]）；只 PTQ INT8 | — | teacher 只能自训 clean（soft_label hook 在，#7 未来） |

- loss：species CE / class-balanced CE + genus aux + family aux + label smoothing。
- **增强**（承实验1 augment.py）：亮度/对比度、轻模糊、JPEG/H.264、随机遮挡、夜视灰度、轻缩放平移、水平翻转、
  退化强度可调；**慎用强 Hue/Saturation**（羽色是关键特征）。检测式裁框抖动（crop_scale_min/crop_ratio 已支持）。

## 4. 类不均衡（长尾，框架已支持）

- 数据侧：`max_per_class` 控盘（01 §9）；端侧只放 ≥阈值的种，<50 张 → cloud_only。
- 训练侧（`configs/train/classify/config.yaml`，二选一）：`optim.class_weighted`（反频 CE）/
  `data.balanced_sampler`（WeightedRandomSampler）。**验证集维持真实分布**（不重采样）。

## 5. 层级输出 + 置信门控（级联第二段）

```
crop → Lite0 →  高置信        → 直接出 species + top-5
                中置信/近似种 → 留云 API 锚点（不实现云端）
                低置信/不在区域清单/稀有 → 回退 genus / family / unknown（不强猜）
```
- 端侧常见种、云端长尾/罕见/疑难。机制在 `cascade/pipeline.py`（置信阈值 + min box + species_conf 门控已就位）。
- 输出结构：`{species, scientific_name, genus, confidence, decision: accepted|cloud|fallback, top_candidates}`。

## 6. 评估口径（承实验1：INT8 数才算数）

1. **主指标**：自有 **test split** top-1/top-5（防泄漏切分，同分布留出）。
2. **INT8**：ORT-QDQ 本地模拟（方向）→ ACUITY 真机（真值）。实验1：lite0 INT8 仅掉 0.19pt。
3. **区域增益**：全局头 with/without mask 对比 + 不加过滤基线（§2，实验1 +1.3pt）。
4. **泛化**：自建跨区域/跨源 holdout（替代 NABirds/CUB，合规）。
5. **困难分组**：近似种混淆、低质量/unknown、(未来)feeder test、夜视/运动模糊/局部身体。
6. **设备指标**（待板子）：NPU 延迟、端到端延迟、内存、模型大小、INT8 掉点、每小时误触发。
> ❌ NABirds/CUB **不用于任何评测**（[[ADR-0005]]，非商用条款覆盖"为商用产品评测"）。

## 6b. 训练监控 + 框架补齐时机

**训练监控**（分类是**我们自己的 Lightning**，不像检测的 NanoDet 子进程——logger 我们掌控）：
- Lightning 逐 epoch：train/val loss + **val_top1 / val_top5**（`module.py` 已 `self.log` prog_bar，控制台实时）。
- **曲线**：接 Lightning 自带 `TensorBoardLogger`（零额外服务、免 aim 在 py3.8 的 hang 坑）；aim 仍可选（`track.aim=true`）。
- **总表**：`eval/ablation/runner.py` 写 `ablation.csv/md`（多 backbone/输入尺寸消融对比，同实验1）。
- **INT8 envelope**：`eval/envelope.py` 出 fp32→int8_sim→field→regional 四级（实验1 同款，INT8 数才算数）。
- 对比检测：检测看 NanoDet 自带 TensorBoard + 总表；分类看 Lightning TensorBoardLogger + 总表（同形态）。

**框架补齐时机**（和检测一样"先抽象、再各源、再下载"）：
- **已在**（实验1 跑通）：Classifier/DataModule（含 `class_weighted`/`balanced_sampler`）、`prep.py` 多源、
  Taxonomy seam、`regional.py`、`cascade/`、envelope/ablation、ONNX 导出 → 分类训练**主干现成**。
- **现在可做**（CPU/设计稳定，与检测训练并行）：`adapters/classify/` 抽象（01 §3b）+ `regional.py` soft-prior
  扩展（01 §7）+ genus/family 辅助 loss + `TensorBoardLogger`。
- **等研究落地**：iNat/GBIF 各源 adapter 精确字段（license 占比/元数据 license 核验中）。
- **等启动分类数据**：上 AutoDL 逐图过滤下载 → 合规并集 manifest → Lite0 训练（排在检测训练之后，GPU/带宽）。

## 7. 端云部署 + 量化

- **端侧**：Lite0 INT8 `.nb`，常见种；**云端**（留锚点不实现）：长尾/罕见/疑难，可用自训 clean teacher（未来）。
- **OTA bundle**（[[ADR-0003]] / plan §C.5）：`net.nb + taxonomy_map(ver) + regional_mask(ver) + min_runtime_abi` 原子下发。
  区域 mask 单独 OTA → 换地区/季节不重训不重发模型。
- **量化**：PyTorch → FP32 ONNX → onnxsim(静态 shape) → ACUITY/pegasus **PTQ**（asymmetric uint8 + 校准集，
  校准集含夜视/噪声/压缩，优先真机采）→ `.nb` → VIPLite。无官方 QAT 路径，先 PTQ。

# 分类 round 选型调研简报 · V861（性能优先重做）

> 背景：分类段像检测 round 一样**扎实重做**。目标芯片从 **V85x（0.5–1T）升级到 Allwinner V861（1 TOPS INT8）**，本轮**性能（精度）优先**、内存/量化难度优先级下降（但"性能相近时其它变量同样计入裁决"，同检测段选 NanoDet 先例）。
> 本文合成三路调研：① V861 芯片能力包络（一手 datasheet + web 确认）② backbone 选型全景 ③ 方法层杠杆 + teacher 许可。调研于 2026-07-05。
> 关联：[[docs/classify/00-螺旋harness起始.md]]、ADR-0001/0002/0005、`results/classify/`（上一轮 Lite0 闭环）。

## 一、V861 芯片能力包络

一手 datasheet（`V861_Datasheet_V0.90, 2026-02`，本地 `/Users/bamboo/Desktop/bird/`；寄存器级细节在另一份 `V861_User_Manual`，暂无）+ web 交叉确认：

| 维度 | 结论 | 来源 |
|---|---|---|
| **NPU IP** | = V853 同一 **VeriSilicon Vivante VIP9000PICO** 血脉的迭代；不是换供应商。真正代际升级在 **AI-ISP 画质**，NPU 只是 0.5→1T 小升级。⚠️**辨析：Allwinner V 系列 NPU 非同一 IP——V831=NVDLA 衍生、R329=ARM China AIPU，只有 V853/V85x/V861 是 Vivante VIP。别拿 V831 的 NCNN 工具链/坑套 V861** | datasheet + V853 逆向确证 |
| **精度** | 硬件 **UINT8 / INT8 / INT16**，**无 FP16/BF16 通路**（ACUITY 广告的 BF16 是满血 VIP9000，PICO 档没有）。分类走纯 INT8 PTQ，INT16 仅作个别敏感层混合精度回旋 | datasheet「1 TOPS @ INT8」+ V853 NPU 文档 |
| **算子** | 官方仅「compatible with common CNN operators」，**未承诺 transformer/attention，无逐算子清单**。conv/dw-conv/pool/add 硬件加速；**SE / h-swish 须实测是否软回退**；**transformer/attention/LayerNorm 继续避开**（软回退吞吐崩塌） | datasheet（三处一致）+ VIP 架构双算子结构 |
| **工具链** | 不变：FP32 ONNX → **ACUITY/pegasus PTQ → .nb → VIPLite**；上层 Tina Linux（OpenWrt/Linux6.6）。**现有 acuity_packager 薄胶水可复用** | Radxa ACUITY 文档 + V853 实操 |
| **内存** | 两档订购型号：**V861M2 = 64MB DDR2 / V861M3 = 128MB DDR3L**，均为**与 4K 编码+AI-ISP 共享的系统总内存**（AI-ISP 官方占 ~64MB）。性能优先选 M3。**可外挂 DDR → 内存非绝对物理墙**，但芯片同时跑检测+ISP+编码+可能其它算法，**"一个算法不能独占太多"是资源克制原则**（呼应 CLAUDE.md §1"资源占用须克制"，用户 2026-07-05） | datasheet Table 2-1 + 用户 |
| **吞吐**（V853 同级实测）| MobileNetV2 156fps / ResNet-18 75fps / **ResNet-50 ~25fps** / YOLOv5s 16fps。分类是级联 crop 后**偶发触发**（非逐帧），单次 50–100ms 预算现实 → 可承受 **~1.5–2.5 GMACs** | neucrack V853 实测 |

**净包络**：**算力不是墙**（分类偶发触发，ResNet-50 级都够）；**内存是"克制墙"**——物理上可外挂 DDR、非绝对上限，但分类算法应克制占用（不独占共享内存），INT8 权重压 **~10–15MB** 为宜（安全区 <12MB），20MB+ 起要么抢内存要么显得独占过多；**算子约束不变**（纯 CNN 安全区、避 transformer、后处理留 CPU、静态 shape）。→ **CLAUDE.md §4 四条硬规则一条不改；上板段仍是最大工程盲点**（SE/h-swish 是否软回退、真机 NPU 与 AI-ISP 争内存后的实际吞吐，都要上板 spike 实测）。

> ⚠️ 决策提示：芯片升级触及"承重规格"（CLAUDE.md §1 仍写 V85x/0.5T）——建议开 **ADR-0008 记录芯片升级 V85x→V861** 再同步 CLAUDE.md，勿擅改。

### 算力落位（1T INT8，~1.5–2.5G 实时预算）
- **稳吃**：Lite2/3、MNV4-Conv-M、RepViT-M1.1、EfficientViT-B1。
- **边界可吃（需实测利用率）**：**Lite4(2.64G)、RepViT-M1.5(2.3G)、MNV4-Conv-L@256(~2.5G)、ConvNeXt-nano(2.5G)**。
- **慎入**：EfficientNetV2-S(8.8G)、ConvNeXt-tiny(4.5G)。
→ 从 Lite0(0.41G) 往上，性能优先可安全升到 **Lite4/RepViT-M1.5/MNV4-L 这一档（~2–2.6G）仍属实时**，IN-1k +6~8 点。

### 输入分辨率
细粒度对细节敏感、且管线已由检测器裁 **紧致 bird crop** 放大喂分类器 → 升分辨率收益比整图更直接。经验：MNV4-Conv-M **224→320 约 +1.5~2 点**（算力 0.65→1.1G）。**256–288 是性价比拐点**，320 以上递减。→ "升分辨率到 256/288" 作为低成本涨点项，与"换更大 backbone"并列做消融，叠加时别双双越 ~2.5G。

### 量化要点
- **Lite 全系最稳**（去 SE、swish→ReLU6，INT8 掉 0.2–0.8%），且与现 baseline 同族、ACUITY 路径复用。
- **RepViT / 重参化纯 conv**：融合成单路卷积、分布规整，PTQ 友好。
- **MobileNet 系 depthwise 必开 per-channel 量化**（per-tensor 会显著掉点）。
- **含 attention/SE/LayerNorm 的量化敏感**（softmax/GELU 后激活非高斯有离群值，朴素 8-bit 常掉 >1%，且板端 pegasus 未必支持 PTQ4ViT 类技巧）。

### 四维加权裁决（命门=鸟种 top-1；差距 ≲1–1.5 点视为"打平"，由次级维度定夺）
参照检测段 NanoDet 先例。四条次级维度按本项目现实排序：**① NPU 算子成熟度**（纯卷积 > 含SE/h-swish > 含attention）**② INT8 量化掉点**（Lite系去SE/swish -0.2~0.8% ≈ 重参化纯conv > 含SE > 含attention常>1%）**③ 体积/克制占用 ④ 工具链落地成熟度**（EfficientNet-Lite 同族已验证 > MNV4/RepViT timm有权重导出干净但端侧量化社区验证少 > ViT 量化需专用技巧）。

### 分层推荐
- **保守档（主推）= EfficientNet-Lite4@256**（Apache/81.5%/+6.4点/INT8**掉0.2%**/四维次级全项最优/同族 pipeline+ACUITY 路径复用）；13MB 属克制上限，若 activation 实测越界退 **Lite3@280**（8.2MB/79.8%/+4.7点）。
- **激进档 = RepViT-M1.5@224–256**（Apache/82.5%/14MB/重参化纯 CNN 无 attention 风险）——**仅当 Lite4 的 81.5% 实测不达命门时才升**，且④工具链成熟度低一档、须先过 ACUITY 量化实测。
- **体积优选 = MNV4-Conv-M@320**（Apache/80.9%/**仅9.7MB**/@320 对细粒度有额外增益）——与 Lite4 几乎打平时，若内存吃紧选它。
- **降级**：~~MNV4-Conv-L(33MB/83.4%)~~——分数虽最高，但**独占内存过多、违资源克制原则**，降出主线。
- **排除**：FastViT、MobileOne（Apple 非商用）、ConvNeXt-V2（CC-BY-NC）。
- **含 attention（EfficientViT/EdgeNeXt/MNV4-Hybrid）**：datasheet 未承诺 attention 算子、无 INT16/BF16 兜底 → 降为"仅 User_Manual 确认支持+量化实测过关"的条件备选。

**"打平选谁"明确裁决**：Lite4(81.5) vs RepViT-M1.5(82.5) 差1点按打平 → **选 Lite4**（工具链同族已验证、量化确定性高）；Lite4 vs MNV4-Conv-M@320(80.9) 几乎打平 → **看内存**（紧→MNV4-M 省3.3MB，够→Lite4）；纯 CNN vs 含 attention 打平 → **一律纯 CNN**。

## 三、方法层杠杆 + teacher 许可

### 三根可落地杠杆（都能挂 Lite0/Lite4 级 student、不碰 transformer 红线）
1. **Decoupled-KD（DKD）**：KD 损失拆 target/non-target 分别加权 + 注意力引导，已验证落 ShuffleNet/Lite 级 student 提分。有 teacher 时的首选提分手段。低分辨率变体（DADKD）正对症"crop 分辨率偏小"。
2. **层级 genus/family aux loss**：把 plan-v2 已定的**层级回退（属/科/bird）从纯后处理升级为训练期 aux CE（HXE 分层交叉熵/taxonomic loss）**——零推理开销、与产品输出同构、让低置信时的层级预测本身更准。**高价值低成本**。
3. **地理先验：硬 mask → SINR 式连续占据概率 + 月份软权重**（推理期重排，不进 NPU 图，与"后处理留 CPU"一致）。RealBirdID(2026) 证明 range-map 使信息增益 57→88、属级 AUC 近满分；但强调是**推理期候选约束、非进模型**——这正是本项目已在做的（GBIF occurrence mask + 物候），升级点是把二值 mask 换成连续软权重更平滑。

### teacher 许可现状（对 ADR-0005 C7 的复核）
| Teacher | 模态 | 权重许可 | 含 NC 数据 | 商用蒸馏 |
|---|---|---|---|---|
| BioCLIP 1/2 | 视觉CLIP | MIT | **是**（TreeOfLife 混 CC-BY-NC）| ❌ 按 §4"NC 传染/换head不洗白"判红 |
| SpeciesNet(Google) | 视觉 | **Apache** | 否 | ⚠️ 许可干净但**粗粒度**相机陷阱分类器，够不到几百鸟种 |
| Perch 2.0(Google) | **音频** | Apache | 是（Xeno-Canto NC）| ⚠️ 音频、且 NC 疑虑；仅关联 plan §9 可选音频 |
| BirdNET | 音频 | CC-BY-NC | 是 | ❌ 已知红线 |
| Merlin(Cornell) | 视觉+音频 | 闭源 | — | ❌ 拿不到权重 |
| iNat21 timm 权重 | 视觉 | 码Apache/权重杂 | 是 | ❌ 数据 NC 污染 |

**结论 = ADR 现状不变**：外部鸟类视觉 teacher 在从严红线下基本不通。**许可干净的替代路** = 借 BirdCLEF 反复验证的 **Noisy-Student 自蒸馏**（自训大模型/集成当 teacher → Lite student），DKD 与层级 aux 都能无缝挂上。若想动 BioCLIP，须先开 ADR 专门裁定"NC 权重传染"这条法律未决问题——**不建议默认放行**。

## 四、对 roadmap / 命门的含义

1. **选型变量升级**：这次性能优先的核心杠杆 = **backbone 从 Lite0 升档 + 升分辨率 256/288**，预期比上一轮 0.748 有可观空间。对应检测 round1 的「bake-off 定选型」——**建议第一圈跑三候选消融矩阵**（各一次训练 + ACUITY INT8 掉点 + 算子回退检查 + activation 峰值内存实测）：**① Lite4@256（主推，验同族路径的精度天花板与内存）② MNV4-Conv-M@320（体积最优+高分辨率增益）③ RepViT-M1.5@224（激进档精度天花板）**，跑完按"命门 top-1 + 四维加权"定档。
2. **命门候选**（供定）：级联提交精度@回退率 / 地域内 top-1 / **层级回退后可用率**——后者因"层级 aux loss 可训练"而更有吸引力（与可训练结构+产品输出同构）。
3. **方法杠杆入队**：DKD 自蒸馏、层级 aux loss、SINR 地理先验——按性价比可排进 round2 富训。
4. **遗留债仍在**（审计）：taxonomy 仍 IdentityTaxonomy 占位（eBird 规范键/regional 有效性未真生效）、iNat 许可未决（红线级）、360 类真实模型未进 registry、gate 阈值全 null。

### 需实测才能收口（上板 spike）
- V861 NPU 是否原生支持 attention → 决定混合架构能否进候选。
- SE / h-swish / GELU-LayerNorm 是否回退 A7 软算子 → 决定 EfficientNetV2/GhostNetV2/ConvNeXt 系实际延迟。
- 建议 spike 里对 **Lite4 / RepViT-M1.5 / MNV4-Conv-M** 各跑一次 INT8 掉点 + 算子回退检查再定档。

### 需开 ADR
- **ADR-0008**：芯片升级 V85x→V861（承重规格变更）+ 同步 CLAUDE.md §1。
- （若要动 BioCLIP）ADR 裁定 NC 权重传染。

# ADR-0001：可行性优先 — 无板子下离线验证细分类器能否达标

- 状态：Accepted
- 日期：2026-06-19
- 相关：`docs/plan-v2.md` §5.3/§5.4/§8、`docs/engineering.md` §5.5/§7、`CONTEXT.md`

## 背景

暂无 V85x 板子，但当前最关心「**微调后的模型能不能达成目标**」。该问题的核心是**精度**，可在 dev/GPU（AutoDL）离线回答；上板只决定延迟/内存/INT8 实测。需要先钉死「目标口径」与「类规模策略」，否则实验设计无从下手。

## 决策

1. **可行性优先、分类器先行**：先建「可行性包络」（FP32 → −INT8(ORT-QDQ 模拟) → −domain gap(退化增强) → +地域过滤）离线估计现场精度区间。先做细分类器（最难项），检测器随后。
2. **类规模 = 全局头 + 地域消融接口**：训练数据覆盖的最大类集训一个全局分类头；地域清单做**推理期 mask**，作消融对比（on/off 都测）。不预设固定大类数（plan §5.3 数据驱动）。
3. **达标口径 = 先看包络再定数**：不预设硬指标；先产出逐级掉点表 + 现场区间，再据此定 gate 阈值或决定加数据/缩类。
4. **开发顺序按 plan 搭框架**：开发阶段先把框架（含数据准备模块）按 plan/engineering 搭好；「数据 ready 是第一步」指**运行时**顺序，不是「先下完所有数据再写码」。用本机已有数据集当开发夹具。

## 理由

- 地域过滤是 plan 认定的「最大精度杠杆」，而它本就是推理期 mask → 「全局头 + 消融」天然同时给出全局与地域两个数，比一开始锁区域更灵活、且能量化增益。
- 无板子下，ORT-QDQ 是 ACUITY 将消费的同形态 ONNX，INT8 掉点方向性可信；退化增强是 domain gap 的合理代理（但非真现场，需诚实标注）。
- 先定硬数会变成拍脑袋；包络让数据说话。

## 影响 / 后果

- 训练侧需支持「全局 N 类头」+ 评估侧支持「加载地域 mask 重算指标」。
- 评估必须三口径并列（验证集 / 模拟 INT8 / 类现场），且**显式标注「类现场 ≠ 真现场」**，避免 99.5%→88% 式误判。
- 当前数据仅作可行性，**产出权重不得进商用**（版权未核）；商用安全数据集是并行的后续轨。
- 检测器、音频、上板暂缓（见 CONTEXT「当前不做」）。

## 备选与放弃理由

- **一开始锁定单区域几十类**：可行性最高，但丢掉「量化地域增益」的能力，且过早绑死市场；保留接口即可，不必现在锁。
- **先定硬指标当 gate**：无数据支撑会失真；改为包络出来后再定。

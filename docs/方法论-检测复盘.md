## 二、贯穿始终的方法论（10 条，每条有实例）

1. **单一命门指标，不被整体指标带偏** —— 用 bird AP50 而非整体 mAP。bake-off 里 RTMDet 整体 mAP 更高但把 bird 稀释了，按命门选 NanoDet。（"整体 mAP 不直观"的活例）
2. **可行性优先：先建掉点包络，不预设硬指标** —— 级① FP32 → ② INT8 PTQ → ③ 域 gap → ④ 地域增益 的逐级掉点链；先答「能不能达标」再定硬数（[[ADR-0001]]）。
3. **过程代理 vs 最终定案分离** —— val 峰只作过程代理，**固定 held-out test 才定案**；scaling 读数一律看 test，绝不拿 val 出货。
4. **scaling/消融量化取舍，不拍脑袋** —— 数据量 + 参数 + 分辨率三扫，甜点是**扫出来的**（1.0x@416：1.5x 不涨、0.5x 掉 4pt、50% 数据近饱和），不是猜的。
5. **证据驱动，不猜** —— decode 喷框查根因（非猜）；移动端 C++≡Python 差异用受控实验隔离（先排 fp16 → 实为 clamp/NMS 坐标空间不一致 → 对齐后 bit 一致）。
6. **诚实 caveat + 正确定位** —— 非鸟弱环 / val→test 落差 / small 目标不做 / worksite 乐观上界（含泄漏），全部披露并正确定位（设计取舍 ≠ 缺陷）；**test 泛化 vs worksite 部署两个数并读**。
7. **两目标 / 两尺子各自分离** —— 部署：Vivante 边侧**剥 sigmoid** 出 logits vs 移动端**焊 sigmoid + 归一化**（喂图出框）；评测：held-out test（泛化）vs worksite（部署）。
8. **复用 OSS + 薄胶水 + 上游只 FP32** —— NanoDet 上游、只在量化落地写薄自研胶水；上游只产 FP32 ONNX，INT8 交板端（§4.1）。不 fork 改框架源码。
9. **许可红线全程 + 逐图署名** —— 数据只用 CC0/CC-BY/自采、避 NC/GPL/AGPL、teacher/data NC 传染意识；**收尾做许可审计**（round2 全训练集 CDLA-Permissive + CC-BY-4.0，无 NC/GPL，可商用）（[[ADR-0005]]）。
10. **不可逆决策 → ADR；每步产物落盘可追溯** —— ADR-0001..0007；results/ + DVC（模型）+ memory + git PR + 飞书报告，逐步可回溯。

## 三、一句话提炼

> **命门定指标 → 零样本证可行 → bake-off 定选型 → 根因诊断落 ADR → 富数据训 + scaling 扫出甜点 → 三尺子诚实评测 → 剥/焊两目标出货 → 许可审计 + 落盘收尾。**

底色三条：
- **verify 心态**：默认「有待改进」，受控实验定论（不是 ok 就信——本轮 str_replace 返回 ok 但没生效、"table cell 粗体没渲染"实为误报，都是靠复核发现）。
- **诚实**：弱环披露、乐观/真值并读，报告框成设计取舍而非藏缺陷。
- **可追溯**：每阶段产物落盘（report / ADR / memory / git / 飞书）。

## 四、套到下一段（分类，待定）

分类种类多、有地域先验、端云回退更复杂，**更该按此方法论做**。可直接复用的骨架：
- **命门指标**：先定分类的命门（如地域清单内 top-1，或层级回退后的"可用率"），别只看全局 top-1。
- **可行性包络**：FP32 → INT8 掉点 → 域 gap（现场退化增强）→ 地域过滤增益，先建区间。
- **过程/定案分离**：val 训练指标 vs 固定 field/现场测试集定案。
- **消融量化**：模型族 / 输入尺度 / crop 策略 / 数据量 scaling 扫甜点（分类 v1/v2 crop 消融已起步）。
- **诚实评测**：域内 vs 现场（field）两个数并读（分类此前 field top-1 0.37→0.63 就是这个套路）。
- **端云 + 层级回退**：低置信/不在地域清单/稀有种 → 属/科/bird 回退 + 云锚点（[[ADR-0002]] eBird seam）。

## 1. 摘要与结论（先给最重要的）

本次实验首次把全链路从数据准备一路打通到级联端到端评估，并完成了一轮针对级联掉点的优化对照。**它不是要刷 SOTA，而是要回答四个可行性问题，并诚实标出每个数字的边界。**

### 1.1 五条决定性结论

| # | 结论 | 关键数据 | 为什么重要 |
|---|---|---|---|
| **C1** | **分类器选 `efficientnet_lite0@224`（不是 val 头名 mobilenetv3）** | int8 后 eff_lite0 反超 mobilenetv3 **2.37pt**（0.9193 vs 0.8956）；mobilenetv3 int8 掉点是 eff_lite0 的 **~20 倍**（3.71pt vs 0.19pt） | V85x 是 INT8-only，**ship 的就是 int8 这个数**，fp32 领先无意义。坐实 §4：mobilenetv3 的 h-swish+SE 量化不友好。 |
| **C2** | **两段都对 INT8 PTQ 友好（方向性）** | 分类 int8 掉点 0.19pt（top-1）；检测 int8 掉点 0.19pt（mAP）、1.96pt（AP50） | 契合 0.5T/INT8-only 的 NPU；上游 FP32→端侧量化的路线在精度上站得住。 |
| **C3** | **级联端到端会掉 ~6.3pt，且 int8 检测额外再掉 8.3pt——这是 COCO mAP 完全测不出来的** | 级联 fp32 = 0.8583（vs 分类直评 0.921）；int8 检测级联 = 0.7750，但检测 mAP 只掉 0.19pt | **mAP 严重低估 int8 对级联的真实伤害**；级联里**检测框质量 >> 检测 mAP**，端到端才暴露。 |
| **C4** | **基线即最优；级联掉点无法靠重训分类器补回** | A1/A2/A2v2/B 四个优化方案均未超过基线；A2v2（更强分类器 val 0.947 + 见过裁框）级联仍 0.860≈基线 | 6.3pt 主要是**裁框丢失细粒度判别信息（非可学 domain shift）+ birds525「整图=鸟」的评估假象**，不是分类器能学回来的。 |
| **C5** | **真正提升级联的方向在检测侧 + 真实数据，不在分类器** | bird AP50 仅 61.6、AP_small 仅 0.165；真实喂食器 bird 检出率仅 **64.5%**（vs 干净鸟照 100%） | 粗检测器的 bird 召回是命门；下一步应投在检测框质量 / 小目标 / 带 species 标注的真实喂食器数据。 |

### 1.2 一句话总览

> **可行性成立**：双段级联（NanoDet-Plus 粗检测 → eff_lite0 细分类）在 INT8 约束下精度可接受、链路可跑通、量化方向友好；**但级联端到端暴露了 mAP 测不出的「检测框质量」命门**，这是后续投入的真正杠杆点，而非分类器重训。

### 1.3 已发布

`efficientnet_lite0@224` 可行性权重已发布到 **registry stable channel**：fp32 93.3 / int8 92.5 / field 81.2。**注意**：此为可行性权重（taxon_key 仅映射 408/525、数据集含许可受限源的同源），**不进商用发行**。

## 2. 实验设置

### 2.1 硬件与分卡

NanoDet 上游 pin `torch<2.0`，无法在 RTX 5090（Blackwell，sm_120，须 torch cu128）上跑，故双卡分工：

| 卡 | 角色 | 任务 | 软件栈 |
|---|---|---|---|
| **RTX 5090**（westd seetacloud） | 分类 | birds525 分类训练 + 消融 + 包络 + 级联评估 | torch 2.10.0+cu128（CUDA 12.8） |
| **RTX 3080**（nmb2） | 检测 | NanoDet-Plus 全量微调 + ONNX 导出 + int8 | base env py3.8 + torch 1.11+cu113 |

### 2.2 环境/平台铁律（§4）

本实验严格遵循 CLAUDE.md §4 不可触碰的硬规则：

1. **上游只产 FP32 ONNX**，绝不上游 INT8。
2. **检测后处理（NMS/decode/grid/sigmoid/anchor）留 A7 CPU**，导 ONNX 时切「裸 backbone+head」。
3. **复用 = 依赖上游 + 扩展点**，不 fork 改框架源码（NanoDet 半停滞锁版属唯一例外）。
4. 许可红线（海外发行从严，避 AGPL/GPL/CC-BY-NC）。

### 2.3 量化的诚实边界（贯穿全报告）

> **本实验所有 int8 数据均为 ORT-QDQ 模拟，是方向性参考，不是板子真值。** 板上真值来自 ONNX → ACUITY/pegasus PTQ → `.nb` → VIPLite/awnn。ONNX 仅作上游交换格式。凡引用 int8 数字处，结论只取**掉点趋势/相对大小**，不取绝对值。

| repvgg_a0 | 224 | 0.9206 | 0.9657 |

val 头名是 **mobilenetv3@224**，仅比 eff_lite0@224 高 **0.28pt**。两强（mobilenetv3@224、eff_lite0@224）进 test 全包络对比。

### 4.2 最优候选 test 全包络对比（含纯 int8 掉点）—— 决定性翻盘

额外加 `fp32_test`（排除 val→test 分布差），int8 掉点 = `fp32_test − int8_sim`（同 test 集，干净对比）：

| backbone | fp32_val | fp32_test | int8_sim | int8 掉点 | field | field 退化 |
|---|---|---|---|---|---|---|
| efficientnet_lite0 | 0.9334 | 0.9212 | **0.9193** | **0.0019（0.19pt）** | 0.809 | 0.1122 |
| mobilenetv3_large_100 | 0.9362 | 0.9327 | **0.8956** | **0.0371（3.71pt）** | 0.8436 | 0.0891 |

**翻盘逻辑**：

- fp32 下 mobilenetv3 领先 1.15pt（0.9327 vs 0.9212）。
- 但 mobilenetv3 int8 掉点 **3.71pt**，是 eff_lite0（0.19pt）的 **~20 倍**。
- 量化后 **eff_lite0 反超 2.37pt**（0.9193 vs 0.8956）。
- V85x 是 INT8-only → ship 的就是 int8 这一档 → **efficientnet_lite0 是明确赢家**。

#### 图② 分类量化对比（最关键）（图表数据）

fp32_test vs int8_sim：

| backbone | fp32_test | int8_sim | INT8 掉点 |
|---|---|---|---|
| eff_lite0@224 | 0.9212 | 0.9193 | 0.19 pt |
| mobilenetv3@224 | 0.9327 | 0.8956 | 3.71 pt |

> mobilenetv3 fp32 更高，但 INT8 后 eff_lite0 反超 2.37pt。V85x 是 INT8-only → eff_lite0 赢——坐实 §4：mobilenetv3 的 h-swish + SE 对 INT8 PTQ 不友好（激活分布长尾、逐通道敏感）。

> **机制坐实 §4**：mobilenetv3 的 **h-swish + SE** 对 INT8 PTQ 不友好（激活分布长尾、逐通道敏感）。这正是 §4「避坑算子」预判的现实。
>
> （旁注：mobilenetv3 在 fp32 下 field 鲁棒性更好——field 退化 8.91% vs eff_lite0 11.22%——但端侧不相关，因为端侧跑的是 int8。）

### 4.3 四级可行性包络（首跑，eff_lite0@224）

来自 `00_首次单模型闭环/report.json`：

| 级 | top-1 | top-5 | vs fp32 | 说明 |
|---|---|---|---|---|
| fp32_val | 0.933 | 0.976 | — | 干净验证集 |
| int8_sim | 0.925 | 0.971 | −0.008 | ORT-QDQ 模拟 INT8，掉点极小（量化友好） |

> **regional 级 0.417 是 artifact，已修正（见 §6.5）**：现行口径在全球 525 类 test 上加区域 mask，把非美国种真值压 −inf → 必错，**并非真实回归**。改在 in-region 子集比 on/off 后，地域过滤真实效果 = **+1.3pt**（0.9204→0.9336）。41.7% 是 artifact，不作结论。

### 4.4 分类器深度分析（eff_lite0，test n=2539）

整体 top-1 = 0.921 / top-5 = 0.972。

**最差类全是小样本类**（n=3~7）：

- `SPOON BILED SANDPIPER` 0.00（n=4）
- 多个类 0.50

**易混对 = 分类学近似种**（模型行为合理，错在「长得像」）：

| 真值 | 被误判为 |
|---|---|
| SPOON BILED SANDPIPER | DUNLIN |
| BAR-TAILED GODWIT | SHORT BILLED DOWITCHER |
| RUDY KINGFISHER | RUFOUS KINGFISHER |
| WHITE TAILED TROPIC | RED BILLED TROPICBIRD |

> **判读**：错误集中在「稀有种 + 近似种」，不是模型崩坏，是长尾数据 + 细粒度本身难。对应产品的分级置信门控：低置信 / 稀有种 → 层级回退（属/科/bird）+ 留云锚点。

## 6. 级联端到端评估

把训练好的检测（NanoDet-Plus）+ 分类（eff_lite0）串成级联，在 birds525 test 子集（n=600，有 species GT，bird 检出 100%）上跑端到端，量化模型一并评。对照：**分类器直评 test top-1 = 0.921**。

### 6.1 核心结果

| 级联组合 | bird 检出率 | 级联 top-1 | vs 直评 |
|---|---|---|---|
| fp32 检测 + fp32 分类 | 100% | **0.8583** | −6.3pt |
| fp32 检测 + int8 分类 | 100% | **0.8583** | −6.3pt |
| **int8 检测** + fp32 分类 | 100% | **0.7750** | −14.6pt |
| **int8 检测** + int8 分类 | 100% | **0.7767** | −14.4pt |

#### 图⑥ 级联端到端 4 组合（图表数据）

基线对照 = 分类直评 0.921（黄色虚线）：

| 级联组合 | 级联准确率 |
|---|---|
| det fp32 / cls fp32 | 0.8583 |
| det fp32 / cls int8 | 0.8583 |
| det int8 / cls fp32 | 0.7750 |
| det int8 / cls int8 | 0.7767 |
| _（基线：分类直评）_ | _0.921_ |

> INT8 检测掉 8.3pt，INT8 分类几乎无损（两两同高）→ 框质量 ≫ mAP（mAP 仅掉 0.19pt）。

### 6.2 三个掉点来源（端到端才暴露）

1. **裁框取景偏移 −6.3pt**：检测裁出的框比 birds525 训练图（整张鸟照）取景更紧/不同 → 分类器 domain shift。**与量化无关**（int8 分类 = fp32 分类同分）。
2. **int8 检测框定位变差，额外 −8.3pt**（0.858→0.775）：但其 COCO mAP 只掉 0.19pt → **mAP 严重低估 int8 对级联的伤害**；框一歪裁出的鸟就偏，被分类器放大成 8.3pt。**级联里检测框质量 >> 检测 mAP。**
3. **int8 分类器几乎无辜**：级联里和 fp32 分类同分（±0.2pt）。瓶颈是检测框，不是分类量化。

### 6.3 Padding 缓解扫描（推理侧扩框 vs 必须重训）

| 框 padding | fp32 检测级联 | int8 检测级联 |
|---|---|---|
| 0% | 0.8583 | 0.7767 |
| 15% | 0.8567 | 0.7933 |
| 30% | 0.8633 | 0.8067 |
| 50% | 0.8667 | 0.8183 |
| 整图 | **0.9167** | **0.9167** |

#### 图⑦ Padding 缓解扫描（图表数据）

fp32 / int8 检测级联随 padding 比例：

| padding | 0% | 15% | 30% | 50% | 整图 |
|---|---|---|---|---|---|
| fp32 检测级联 | 0.8583 | 0.8567 | 0.8633 | 0.8667 | 0.9167 |
| int8 检测级联 | 0.7767 | 0.7933 | 0.8067 | 0.8183 | 0.9167 |

> 推理侧 padding 弱缓解（fp32 +0.8pt@50%，int8 +4.2pt@50%），仅「整图」完全恢复至 0.9167（≈直评）→ 须在训练侧解决裁框信息损失。

- **推理侧 padding 是弱缓解**：fp32 路 +0.8pt@50%；int8 路 +4.2pt@50%（扩边补偿差框，故 int8 受益更大）。
- **只有「整图」才全恢复**（≈直评 0.917）。
- 但 birds525 上「整图=训练图」，真实喂食器场景没有「整图=鸟」的便利 → **结论：推理侧补不回，必须训练侧解决。**

### 6.4 诚实边界

> birds525 是**已裁好的鸟照、非真喂食器场景**。本评估测的是**级联机制 + 裁框取景偏移 + int8 框质量传播**，**不是真实喂食器 domain gap**（需带 species 标注的喂食器图，暂无）。绝对值受子集（600 张，偏易）影响，**掉点/趋势才是结论**。

## 6.5 地域过滤（likely-species mask）on/off 消融 —— 修正 issue#11 artifact

**背景**：plan §5.4 称地域过滤为「最大精度杠杆」——推理期按地区加载 eBird likely-species 清单，不在清单的类 logit 置 −inf（软先验、不重训、可 OTA）。它不动权重，纯推理期开关，因此理应是「免费」增益。

**方法**：`eff_lite0` fp32 在 birds525 test（n=2539）上，加载 US eBird 区域清单（`regions/us.json`，覆盖 228/525 = 43.4% 的类），对比 mask off / on 两种口径。

| 口径 | mask OFF | mask ON | 说明 |
|---|---|---|---|
| 全局 525 类（n=2539） | 0.9212 | 0.4155 | ⚠️ artifact：非美国种真值被 mask 压成 −inf 必错（issue#11），**非真增益** |
| in-region 子集（真值∈US, n=1130） | 0.9204 | **0.9336** | ✅ 真实地域增益 **+1.3pt** |

#### 图⑨ 地域过滤 on/off（图表数据）

两组口径，每组 mask off vs on：

| 口径 | mask OFF | mask ON |
|---|---|---|
| 全局 525 类 | 0.9212 | 0.4155（artifact，灰/红） |
| in-region 子集 | 0.9204 | 0.9336（真增益，高亮） |

> 全局口径 mask ON 的 −50pt 暴跌是**测法错误**（全球 test 加区域 mask 把外地真值压掉），不是地域过滤的实际效果；in-region 口径才是部署场景（鸟确实是本地种）下的真实答案。

**结论（诚实）**：

- **真实答案：加地域过滤 vs 不加 = +1.3pt**（部署场景 = 鸟确实是本地种），且**免费**（推理期、不重训、可 OTA）。
- 全局那个 −50pt 暴跌是**测法错误**（全球 test 加区域 mask 把外地真值压成 −inf 必错），**不是地域过滤的实际效果** —— 这正是 §4.3 / §8 那条 regional 41.7% 的来源，本节予以修正。
- **增益温和的原因**：birds525 模型本就强，mask 只纠正「误判成外地相似种」的少数样本；**部署地相似种更多、或模型更弱时，增益会更大**。

> **修正 §4.3 / §8#8**：四级包络里的 regional 41.7%（0.417）是上述全局口径 artifact，**非真实回归**。地域过滤的真实效果是 **in-region +1.3pt**（0.9204→0.9336）。

## 7. 掉点优化实验（A/B 对照）

承接级联掉点（fp32 取景偏移 6.3pt + int8 检测 8.3pt），试三条路补回。

### 7.1 A：分类器侧（让分类器适应检测裁框）

| 方案 | 级联 p=0（紧框） | 整图 | 取景 gap | 结论 |
|---|---|---|---|---|
| 基线 | 0.858 | 0.917 | 0.059 | — |
| **A1** 通用强裁剪（RandomResizedCrop scale0.4 + 宽 ratio） | 0.855 | 0.907 | 0.052 | ❌ 无改善，还伤干净精度（0.933→0.913） |
| **A2** 纯检测裁框训练（数据少 9199） | 0.803 | 0.848 | 0.045 | ⚠️ 更差（数据少混淆）；但 gap 缩小 → 机制有效 |
| **A2v2** 混合（原图 11750 + 裁框 9699，val 0.947） | **0.860** | 0.912 | 0.052 | ≈基线，精度修复但级联未提升 |

#### 图⑧ 优化 A/B 对比（图表数据）

级联 p0，vs 基线 0.858（黄色虚线）：

| 方案 | 级联 p0 |
|---|---|
| A1 | 0.855 |
| A2 | 0.803 |
| A2v2 | 0.860 |
| 基线 | 0.858 |

> 分类器侧训练（A1/A2/A2v2）均未超基线（0.858）→ 掉点是裁框信息损失 + 评估假象，非可学 domain shift。B 检测延长边际（mAP 0.591→0.594）。

**逐方案解读**：

- **A1**：随机裁剪没对准鸟、跟检测框分布不同 → 证伪「通用/推理侧增强能补回」的猜测，且伤干净精度。
- **A2**：纯检测裁框训练数据太少（9199）导致混淆变差，但**取景 gap 缩小（0.045 vs 基线 0.059）→ 机制其实有效**，只是被数据量拖累。
- **A2v2**：混合训练修复了精度（val 0.947，甚至高于基线），也见过裁框，但**级联 p=0 仍 0.860≈基线**。

> **A 结论**：**分类器侧训练无法补回级联 6.3pt。** 该掉点主要是**裁框丢失细粒度判别信息（鸟被裁紧、区分种的细节没了，非可学 domain shift）+ birds525「整图=鸟」的评估假象**（真实喂食器场景没有整图对照，裁框是必需的，「vs 整图」的 gap 部分是人为的）。

### 7.2 B：检测框质量（延长 416 训练 50→100ep）

|  | 50ep | 100ep |
|---|---|---|
| mAP | 0.591 | 0.594 |
| AP_small | 0.165 | 0.180 |

> **B 结论**：边际提升。**小目标（bird）是瓶颈**，延长训练帮助有限，需 512 输入 / 改架构（增边缘算力，权衡）。

## 9. 结论与下一步

### 9.1 最终结论

**整体结论四要点：**

- **基线即最优**：分类侧所有训练干预（A1/A2/A2v2）均未超过基线 0.858，说明级联约 6.3pt 的掉点来自裁框信息损失 + birds525 评估假象，而非可学习的 domain shift——堆分类器训练无效。
- **INT8 检测的 8.3pt 才是更实问题**：检测 mAP 量化后仅掉 0.19pt，但级联端到端却掉 8.3pt——框质量被裁剪放大、mAP 指标测不出来。这是比分类量化更值得关注的真实痛点。
- **分类量化几乎无损 + eff_lite0 是正解**：eff_lite0@224 INT8 仅掉 0.19pt，且 INT8 下反超 mobilenetv3 2.37pt。V85x INT8-only 平台上，eff_lite0 是细分类器的确定选择。
- **真提升方向**：下一步收益集中在检测侧（bird 召回 / 框质量 / 训练侧裁框补偿），以及引入真实喂食器 species 数据，而非继续优化分类器。

**展开：**

- **基线即最优**：分类 `eff_lite0@224`（test 0.921，int8 −0.19pt）、检测 NanoDet 50ep（mAP 0.591）。A1/A2/A2v2/B 四个优化方案均未超过基线。
- **级联 ~6.3pt 掉点** = 裁框信息损失 + birds525「整图=鸟」评估假象（部分 gap 是人为的）。
- **int8 检测 8.3pt 是更实的问题**：框定位经裁剪放大，且 COCO mAP 测不出。最稳的解仍是**端侧 ACUITY 量化质量 + 检测框质量**，而非分类器重训。
- **方法论收获**：级联里 **检测框质量 >> 检测 mAP**——这是单独评检测时看不到、只有端到端才暴露的关键。

### 9.2 下一步（优先级排序）

1. **检测侧提 bird 检出率 / 框质量**（最高杠杆）：416→512、按小目标调尺度、补鸟数据，攻 AP_small 0.165 与 bird 召回 64.5%。
2. **真实喂食器带 species 标注的数据**：消除「整图=鸟」评估假象，量到真实 domain gap。
3. **板上 ACUITY 真值回灌**：当前所有 int8 为 ORT-QDQ 模拟；真值须 ONNX→ACUITY/pegasus PTQ→`.nb`→VIPLite，并把板上 latency/精度回灌 EvalReport（当前最大工程盲点，engineering.md §8.1）。
4. **skunk 类不均衡**：补数据或并类（当前 AP 1.3）。
5. ✅ **已修 regional 包络口径**（§6.5）：in-region 子集 on/off 对比，地域过滤真实 +1.3pt；后续可扩展更多区域清单 + 部署地实测增益。
6. **可选**：蒸馏 / 更大输入提级联（权衡边缘算力）。

# 级联端到端评估 — 2026-06-20（detect→crop→classify）

把训练好的**检测(NanoDet-Plus)+ 分类(efficientnet_lite0)**串成级联,在 birds525 test
子集(n=600,有 species GT)上跑端到端,量化模型也一并评。回答"合在一起整体推理效果如何"。

## 怎么串的（拓扑 = V851S 部署形态）

```
原图 ─[检测 ONNX]→ 裸head(1,3598,43) ─[CPU decode: GFL分布→bbox+grid+NMS]→ bird框(原图坐标)
                                                                              │
                                          [CPU 裁剪+预处理 resize224+ImageNet归一]
                                                                              │
                                                  [分类 ONNX]→ 525 logits → species
                                                  (无 bird 框则回退整图)
```
NPU 跑卷积(检测backbone+head / 分类网络),A7 CPU 跑 decode/NMS/裁剪/预处理(§4 铁律)。

## 格式
- 检测：FP32 ONNX(opset11) [1,3,416,416]→[1,3598,43] 裸head；+ INT8 QDQ(opset13 per-channel)
- 分类：FP32 ONNX [1,3,224,224]→[1,525]；+ INT8 QDQ
- ⚠️ int8 = ORT-QDQ **模拟**,板上真值来自 ACUITY PTQ→.nb(ONNX 仅上游交换格式)

## 核心结果（对照：分类器直评 test top-1 = 0.921）

| 级联组合 | bird检出率 | 级联 top-1 | vs 直评 |
|---|---|---|---|
| fp32检测 + fp32分类 | 100% | **0.8583** | −6.3pt |
| fp32检测 + int8分类 | 100% | **0.8583** | −6.3pt |
| **int8检测** + fp32分类 | 100% | **0.7750** | −14.6pt |
| **int8检测** + int8分类 | 100% | **0.7767** | −14.4pt |

### 两大掉点来源（端到端才暴露）

1. **裁框取景偏移 −6.3pt**：检测裁出的框比 birds525 训练图(整张鸟照)取景更紧/不同 →
   分类器 domain shift。**与量化无关**(int8分类 = fp32分类同分)。
2. **int8 检测框定位变差 −8.3pt(额外)**：换 int8 检测器,级联 0.858→0.775。但其
   **COCO mAP 只掉 0.19pt** —— **mAP 严重低估 int8 对级联的伤害**;框一歪裁出的鸟就偏,
   分类器被放大成 8.3pt。**级联里 检测框质量 >> 检测 mAP**。
3. **int8 分类器几乎无辜**：级联里和 fp32 分类同分(±0.2pt)。瓶颈是检测框,不是分类量化。

## Padding 缓解扫描（推理侧扩框 vs 重训）

| 框 padding | fp32检测级联 | int8检测级联 |
|---|---|---|
| 0% | 0.8583 | 0.7767 |
| 15% | 0.8567 | 0.7933 |
| 30% | 0.8633 | 0.8067 |
| 50% | 0.8667 | 0.8183 |
| 整图 | 0.9167 | 0.9167 |

- **推理侧 padding 是弱缓解**：fp32 路 +0.8pt@50%、int8 路 +4.2pt@50%(扩边补偿差框,故 int8 受益更大);
  **只有"整图"才全恢复**(≈直评)。
- 但 birds525 上"整图=训练图",真实喂食器场景没有"整图=鸟"的便利 → **结论:推理侧补不回,必须训练侧解决**。

## 结论 → 优化方向
- **#1 高杠杆:分类器用"检测式裁框"增强重训**(A1 调强 RandomResizedCrop;A2 用真实检测裁框/含int8框做训练集)
  → 同时治 取景偏移 + int8差框。
- **#2 检测框定位质量是级联命门**:检测 512/延长/提小目标 AP(bird AP_small 0.165)。
- 实验脚本见 `scripts/`(cascade_eval / cascade_int8 / padding_sweep[_int8] / cascade_prep)。

## 诚实边界
birds525 是已裁好的鸟照、非真喂食器场景。本评估测的是**级联机制 + 裁框取景偏移 + int8框质量传播**,
非真实喂食器 domain gap(需带 species 标的喂食器图,暂无)。绝对值受子集(600张,偏易)影响,**掉点/趋势**才是结论。

# 观鸟器工作场景评测报告（round2 · main 模型）

> **独立评测集**，与 train/test 切分无关、跨 split 全量取图（用户 2026-07-05 定，**不考虑数据泄漏**）。
> 目的：看模型在**产品实际面对的场景**（观鸟器视角、中大目标、全类别 + 空帧）上的表现。
> 产物：`worksite_eval.json`（GT）+ `main_detections.json`（main 预测）+ 本报告。构建脚本 `scratchpad/build_worksite.py`、评测 `worksite_metrics.py`。

## 1. 集构建

- **范围**：train_train + train_val + test_test 三 split 并集（file_name 去重、重编 id）。**故意含训练图**（泄漏），因为产品部署时就是面对这些同款场景。
- **视角过滤**：只取**固定机位 / feeder / 近景**源——camera-trap（`caltech_ct` / `ena24`，固定机位拍自然场景，与观鸟器同构）+ 喂食台（`roboflow_feeder` / `birdv2`）+ 近景（`roboflow_squirreldet` / `csp` / `meproject`）。**排除** `open_images`（网图任意视角）、`inat_md`（微距）。
- **尺度**：只取**中大目标**（area ≥ 32²px），即观鸟器工作点。
- **构成**：bird 主导 + 全类别 + 空帧负样本。

## 3. 读数

- **命门 bird 在工作场景 AP50 94.9、大目标 98.5** —— 部署同款场景下召回极强。
- **全类别 79–95** —— 非鸟类在工作场景也够用（远好于 held-out test 的 51–67），因为工作场景=中大+固定机位，正是各类的强项。
- **背景误报**：conf 0.4 时 7.5% 空帧会误报（0.5 降到 3.3%）。观鸟器会拍大量空帧（风吹触发），**这个假警率产品需正视**——建议部署阈值 ≥0.4 并叠加时序滤波（连续 N 帧才告警）压误报。
- bird medium（69.7）明显低于 large（98.5），再次印证「产品只做中大」。

## 4. Caveat（诚实披露，务必与 held-out test 并读）

本表数值**显著高于** round2 固定 held-out test（bird AP50 85.0 / 大目标 90.1），原因有三，**不可当泛化真值**：
1. **数据泄漏**：跨 split 含训练图，模型对部分样本是"记忆"而非"泛化"（用户明确接受）。
2. **只中大目标**：排除了小/远的难样本。
3. **视角挑选**：只留固定机位/feeder 源，是模型的强项分布。

**定位**：本报告 = **"部署同款场景（含记忆样本）的乐观上界"**；**泛化真值看 held-out test**（`训练监控日志.md §收口`，bird 85 / 大 90）。两者互补——test 答"没见过的场景行不行"，worksite 答"部署这类场景表现多好"。

## 1. 数据装配（① 进行中）

**分配原则**：按产品角色+检测难度（非平均）——bird 命门最多、person 够检到即可、其余够区分。

| 源 | 域 | 供类 | 状态 | 量 |
|---|---|---|---|---|
| **ENA24** | 相机陷阱(东北美) | bird/squirrel/cat/other | ✅ round1 复用 | round1: bird 1620 等 |
| **Caltech CT** | 相机陷阱(美西南)+empty | bird/squirrel/other+负样本 | ✅ round1 复用（CCT 坐标 bug 已修） | — |
| **Roboflow "Bird feeder"** | **feeder 真实域** | 5 种鸟→**bird** | ✅ 下完+审计+label_map 修 | train **3150** bird 框 |
| **OIV7** | 网图 | **person**(cap 3k)+bird(cap 6k) | ✅ CSV 通(SELECT_OK)→图像下载中 | person 3k(13048 框)+bird 6k |
| **iNat Open Data** | 自然特写 | **bird**(MD 伪标注+LS 人审) | 🟡 管线建成→待 box 跑 | 命门主力，多域丰富 |

**分布目标**（§计划 3）：bird ~10–12k（命门最多）· person ~3k · squirrel/cat ~2–3k · other_animal 压 ~4–5k。

### 装配过程记录（坑与解）
- **下载提速**：Roboflow/OIV7 单连接被 CDN 限速 ~24KB/s → **aria2c 16 连接 1.9MB/s（80×）**。大文件偶发某片卡停
  → aria2c `-c --max-tries --timeout` 续传补完。→ 该固化进 acquire（管线打磨）。
- **Roboflow label_map**：占位键(bird/squirrel/...)与真实类目(5 种鸟名)不符 → 会全丢=0 框；audit 后据实
  species→bird（PR #68）。
- **acquire 幂等门 bug（PR #72）**：动态源（s3/roboflow/inat）`archive_sha256` 为空 → `_checksums_ok`
  对空集**空真** → `not True` 误跳 `_fetch` → OIV7 acquire **0 图**。修：有 checksum 且通过才跳，否则必 fetch。
- **OIV7 CSV 大字段**：Python csv 默认字段上限 128KB，OIV6 大 CSV 超 → `_csv.Error`；抬 `field_size_limit`。
- **OIV7 CSV 损坏**：aria2c 并行下**某片卡停留坏 chunk**（字段 >10MB）→ fresh 重下 + 验字段。
- **csv 解析（QUOTE_NONE, PR #74）**：OIV6 bbox CSV 无引号字段，默认 csv.reader 遇杂引号读巨字段 → QUOTE_NONE。
- **★ 根因：系统盘满（30G / 99%）** —— 反复下载损坏 + SSH/命令发飘的**真元凶**。数据下的是数据盘（对），但
  **conda env(23G)+ 下载/安装缓存 `.cache`(8.6G) + `/tmp`(4.2G) 撑满 30G 小系统盘** → aria2c/unzip 写临时失败
  致 corrupt + 整机不稳。**修：清缓存(99%→54%) + `TMPDIR`/`PIP_CACHE_DIR` 重定向到数据盘（写进 .bashrc）。**
  **教训：AutoDL 系统盘只 30G，装 env/下模型前必把缓存+temp 指到数据盘，否则撑满连累一切。**
- **OIV7 CSV 修复验证**：盘修好后 aria2c -x16 重下 2GB CSV（~1MB/s，Google 服务端单文件限速，非本地问题），
  `select_images` 出 `SELECT_OK`：18000 图（bird 15000 / person 3000），person **13048 框** / bird 39134 框
  → 字段/QUOTE_NONE/幂等门/满盘四坑全解，OIV7 person 源通。图像下载收 bird→6k（网图不宜主导多域混合）。
- **screen 重定向坑**：`screen -dmS x bash -c "..." > log` 的 `> log` 重定向的是 **screen 自身**输出（空），
  非内层命令 → log 一直空、误判。改把 `> log 2>&1` 放进 `bash -lc "内层命令 > log"` 才捕获到。
- **★ urllib 在 GFW 下挂死（PR #77）**：iNat demo 卡死在 API 枚举——`urllib.urlopen` 在 GFW 下常在
  **IPv6/DNS 挂起**且 `timeout` 不生效（同 API `curl` 秒通）。修：`_http_get_json/_http_download` 改
  `curl --ipv4 --max-time`（超时可控 + 重试）；`fetch_inat_aves_obs` 加游标防空转（id 不前进则 break，
  绝不翻遍 4M 观测）。**教训：box 上一切外网抓取走 curl --ipv4，别用 Python urllib。**
- **MD 环境复用**：box 已有 round1 的隔离 `md` env（PytorchWildlife 1.3.0）+ 缓存 `MDV6-yolov9-c.pt`
  （MIT，§4 安全）→ 伪标注免重装免重下。整条 pseudolabel 在 `md` env 跑（pydantic/PIL/numpy 齐全）。

### iNat 教师伪标注 + LS 人审管线（建成，`src/edge_cam/data/pseudolabel/`）

数据管线 §3b「MD 高置信自动收 / 中置信 Label Studio 人审 / 低置信丢」的**可跑实现**（填 `inat_md._fetch`
的 NotImplementedError 缺口）。四步 + 纯函数全测（18 例）：

| 件 | 作用 | 测试 |
|---|---|---|
| `inat_fetch.py` | iNat `/v1/observations` API 枚举 Aves(CC0/CC-BY/research/geo)→并行 S3 下 medium 图 | `parse_inat_api_page`/`medium_url` 纯函数 |
| `md_label.py` | MegaDetector 伪标注→COCO（保 score，animal→bird，person 丢） | `build_pseudolabel_coco` 纯函数 |
| `triage.py` | 按框 score 三分：≥0.7 auto(md_pseudo) / [0.2,0.7) 人审 / <0.2 丢 | `triage_by_confidence` 纯函数 |
| `label_studio.py` | COCO→LS 预标注任务（MD 框作 predictions）；LS 导出→COCO(md_human_verified) | 像素↔百分比对称往返 |
| `run.py` | 一条命令串起 + 渲染教师框预览图（绿=高置信/橙=中置信，肉眼直评） | — |

- **为何走 API 而非 S3 全量 dump**：只需数千 bird，全量元数据 tar.gz 数十 GB 解压 join 代价过高；API 按
  taxon_id=3 直接分页枚举，逐照片带 license/作者，正好喂 `select_inat`（复用其过滤）。
- **信任分层落地**：auto 份 → `inat_md_coco.json`（md_pseudo）；人审份 → `inat_verified_coco.json`
  （md_human_verified）。新增 `inat_md_verified` adapter 注册，config 两行并列 → 训练可按信任加权。
- **许可**：照片 license 用 photo 的 license_code（非观测许可，规避 NC 传染），大写归一对上 CC0/CC-BY 白名单。
